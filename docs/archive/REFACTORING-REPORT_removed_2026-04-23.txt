ARCHITECTURAL BENEFITS
======================

  1. MAINTAINABILITY — Smaller files are easier to navigate and modify
     - No single file exceeds ~2,000 lines (was 9,577 for MainActivity)
     - Domain-specific ViewModels make data flow clear

  2. TESTABILITY — ViewModels can be unit-tested in isolation
     - Each ViewModel has explicit repository dependencies via Hilt
     - No implicit coupling through a monolithic ViewModel

  3. BUILD SPEED — Incremental compilation benefits from smaller files
     - Kotlin compiler recompiles only changed files

  4. CODE ORGANIZATION — Clear module boundaries
     - server.js: each route group in its own module
     - ViewModels: each domain (transit, aircraft, weather, etc.) isolated
     - Preferences: centralized in MenuPrefs.kt
