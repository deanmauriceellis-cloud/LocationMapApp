## Silence Reach-Out + Priority Queue

**User direction:** Never have silence. If 5 seconds pass with no POI trigger, expand search and narrate the next best thing. Prioritize: business value first (paying merchants), then historical importance.

**Changes to `NarrationGeofenceManager.kt`:**
- Added `lastEntryEmitTime` tracking — updated on every ENTRY event
- Added `findNearestUnnarrated()` — searches within 500m radius, sorts by:
  1. `adPriority` DESC (paying merchants first — revenue)
  2. `priority` ASC (historical importance — 1=must-hear, 5=filler)
  3. `distanceM` ASC (tiebreaker)
- Added `triggerReachOut(nearbyPoint)` — marks as narrated and emits ENTRY event
- Added `msSinceLastEntry()` for activity-side silence detection

**Changes to `SalemMainActivityNarration.kt`:**
- TTS idle handler: when queue empties, waits 5 seconds, then calls `findNearestUnnarrated()` and triggers reach-out
- `enqueueNarration()` now priority-ordered: in dense areas, merchants and historical POIs get narrated before minor/filler content
- Queue insertion uses `adPriority DESC, priority ASC` comparison

**Result:** Continuous ambient narration — no dead air. Walking through dense downtown, paying merchants get narrated first. Between geofences, system reaches out to nearest valuable POI within 500m.
