## FuzzySearchEngine — type/category search fix

**Problem:** "dentist" found nothing. Root causes:
1. `when { }` in per-token scoring was exclusive — non-name fields (subcat, desc) never scored once any name tier fired
2. Zero synonym coverage — "dentist" never reaches the "dental" in POI names; subcategory and tags are empty for most POIs

**Fix — `FuzzySearchEngine.kt`:**
- Added `TYPE_SYNONYMS` map: "dentist"→["dental"], "lawyer"→["law","attorney","legal"], "gym"→["fitness"], "coffee"→["cafe"], "doctor"→["medical","physician","clinic","health"], + 30 more entries
- Token expansion: each input token is expanded with synonyms before scoring; `distinct()` prevents double-counting
- Non-name scoring made additive: subcat/cat, cuisine, desc, hist, addr always accumulate regardless of name match (name scoring stays exclusive within its own tiers)
- Default `limit` 60→80; `take(8)` tokens instead of 6
