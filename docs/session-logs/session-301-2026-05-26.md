### S301 — Investigation: tour-mode POI marker filtering + "same icon" symptom

Operator on-device (Pixel 8, S299 debug build), on "Dr. K's Tour", reports (a) all POIs visible in tour mode, (b) all POIs share the same icon across categories. Asked if recent sessions changed display logic or if a DB update caused it.

**Screenshot:** `/tmp/lma_s301_tourmode.png` — dense field of "Historical Buildings"/"Historical Landmarks" ghost-style markers on Dr. K's Tour; faint red tour polyline on the right.

**Findings:**
1. **No recent logic change.** S300 was a no-op opener. S299 was graphics-only: `poi_detail_sheet.xml` fitXY→centerCrop, `PoiDetailSheet.kt` banner 0.55→0.20, `PoiHeroResolver` category mappings, 124 hero webps + 40 category scenes. None touch marker rendering / tour filtering / layer toggles.
2. **No DB cause.** `salem_content.db` clean in git, last modified 2026-05-24 (S293). S299 made zero Room/PG/publish-chain changes (per S299 close).
3. **"Same icon" = by design, not regression.** Map markers come from `MarkerIconHelper.dot(this, place.category)` → `poi-circle-icons/<category>.webp` (old witch circle-icons, untouched S299). `GhostResolver` is ONLY in `CollectionSheet.kt`, never the map (the S299 "reuse ghost badges as markers" plan was never coded). In `MarkerIconHelper.kt:187-191`, both `historical_buildings` AND `historical_landmarks` map to the same asset `tourism_history/historic_building.webp` (S216 intentional) — and the view is dominated by exactly those two categories.
4. **Tour mode does NOT filter markers to the tour's collection** (Explore agent, thorough). Marker visibility (`SalemMainActivity.kt:2740-2766`) is gated by: `showAllPoisActive` FAB (overrides all) → else `is_tour_poi` force-visible → else historical-landmark class gated by `histLandmarkOn` layer toggle → else civic gated by `civicOn` → **else `true` (renders unconditionally)** (line 2763). Tour state only changes the layer-toggle DEFAULT keys (`MenuPrefs.histLandmarkPrefKey(tourActive)`): Explore defaults both ON, Tour defaults both OFF. The tour's `collectionPoiIds` (`TourModels.kt:74-79`) is used ONLY for Collection-sheet visit-counts — NEVER for map markers.

**Conclusion (interim, later corrected):** Two independent systems — (1) map markers = layer toggles + POI flags, collection-unaware; (2) Collection/"ghosts" = tour membership, Collection sheet only. The app has never restricted the map to the active tour's POIs.
