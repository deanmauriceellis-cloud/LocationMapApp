## Walk Simulator Route Rebuild

**Problem:** OSRM-generated 352-point route had ~8 out-and-back backtracking segments (dead-end wharves, pedestrian areas). Route started by going NORTH from PEM instead of west on Essex St. Also wandered to Collins Cove (42.513, way too far south).

**Root cause:** OSRM pedestrian routing is fundamentally broken for Salem's downtown — pedestrian mall, Common paths, and waterfront dead-ends confuse it. Multiple attempts with different waypoints all produced backtracking.

**Solution:** Hand-crafted 86-point clean route following actual streets:
- PEM → West on Essex St pedestrian mall (13 pts)
- North through Washington Sq to Salem Common (9 pts)  
- Around Common counterclockwise (11 pts)
- South on Brown/Liberty to Charter St (7 pts)
- East on Charter St past Old Burying Point (6 pts)
- South to Derby St waterfront (8 pts)
- East along Derby to Seven Gables area (12 pts)
- North back through downtown to PEM (20 pts)

**Metrics:** 86 points, 3.4km, ~40min walk, zero backtracking, clean clockwise loop.

**File modified:** `app-salem/src/main/assets/tours/downtown_salem_route.json` — complete replacement

## Multi-Pass Narration System (Salem KB Integration)

**User direction:** When people cross the same path on different days, give them deeper historical content. Salem knowledge base is the shared content engine.

**Architecture:**
- Pass 1 (first visit): `short_narration` — basic intro (existing content)
- Pass 2 (second visit): `narration_pass_2` — historical deep-dive (figures, events, what happened here)
- Pass 3 (third visit): `narration_pass_3` — primary source voice (court records, depositions, period quotes)
- Pass 4+: cycles back to Pass 1
- Falls through: if pass N has no content, uses previous pass

**Content sourced from Salem KB:**
- NPC emotional arcs, bios, documented quotes (Bridget Bishop, Jonathan Corwin, Rebecca Nurse, Giles Corey, Samuel Parris, Cotton Mather)
- Building atmosphere files (Prison Lane Jail sensory detail, Ingersoll's Ordinary, Parris Parsonage)
- Primary source court records and depositions
- Timeline events and faction perspectives

**Schema changes:**
- Added `narration_pass_2` TEXT and `narration_pass_3` TEXT columns to `narration_points` table
- Room DB version 3 → 4
- `NarrationPoint.kt` entity updated with new fields

**Visit tracking:**
- SharedPreferences `narration_visits` stores per-POI lifetime visit count
- `recordVisit(pointId)` called after each narration playback
- `getNarrationForPass(point)` selects Pass 1/2/3 based on `visitCount % 3`
- `init(context)` must be called to enable persistent tracking

**Narration selection flow:**
1. `NarrationGeofenceManager.init(context)` — loads SharedPreferences
2. On ENTRY event → `enqueueNarration(point)` → `playNextNarration()`
3. `getNarrationForPass(point)` checks visit count, selects pass 1/2/3
4. TTS plays selected narration, bottom sheet displays it
5. `recordVisit(point.id)` increments persistent counter
