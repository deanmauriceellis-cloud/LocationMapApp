### Operator follow-on: "Choosing any map overlay doesn't reflect on the map"

Picker + slider worked; the actual osmdroid map showed no overlay because the rendering wasn't wired. Built it tonight after all.

**`OsmdroidHistoricalOverlay.kt`** — new file. Custom `org.osmdroid.views.overlay.Overlay` that draws historical tiles via the existing `TileArchive` class. Computes visible-tile range from `mapView.projection.boundingBox`, iterates x/y, fetches each tile's bitmap, draws via `canvas.drawBitmap(bmp, null, dstRect, paint)` with `paint.alpha = opacityAlpha`. Verbose-debug `OsmdroidHistOv` log line every 5s with frame counter, visible tile range, attempts/hits.

**`applyHistoricalOverlaySelection` wired through**: creates/swaps `OsmdroidHistoricalOverlay` instance, opens a new `TileArchive(file, "Historical-${year}")` when year changes (closes prior), adds overlay to `mapView.overlays` at index 0, mutates `opacityAlpha` from slider, calls `mapView.invalidate()`. Verbose `HistoricalOv` debug log at every state change (open/swap/remove archive, alpha changes, slider persisted) per `feedback_verbose_debug_until_production.md`.
