## Phase 9P.2: importer — DONE

### Source data investigation
- `tools/salem-data/narration-priority-pois.json` has count=307 (Waves 1-3), no narrations
- `tools/salem-data/merged-salem-pois.json` has 848 raw POIs (deduped from 310+157+687 raw input records)
- **`salem-content/salem_content.sql` is the canonical source** — has 815 INSERT statements for `narration_points`, full content (narrations, related_*_ids, merchant_tier, ad_priority, etc.)
- Decision: import directly from the bundled SQL, not the JSON, because the SQL has the most complete column set

### Schema iteration
- Initial schema had 30 columns. Inspection of bundled SQL revealed I was missing 12 columns: `voice_clip_asset`, `geofence_shape`, `corridor_points`, `related_figure_ids`, `related_fact_ids`, `related_source_ids`, `action_buttons`, `merchant_tier`, `ad_priority`, `custom_icon_asset`, `custom_voice_asset`, `custom_description`
- **`merchant_tier` and `ad_priority` are foundation fields for the future advertising business** — preserving them now is critical
- Dropped the empty table, expanded schema to 42 columns, re-ran migration. Verified clean.

### Importer architecture
- Created `cache-proxy/scripts/import-narration-points.js`
- Uses `better-sqlite3` (already a cache-proxy dep) to load extracted INSERTs into in-memory SQLite, then reads rows and UPSERTs into PostgreSQL via `pg`
- 6 steps: extract → load to SQLite → read rows → connect PG → upsert → verify
- Idempotent: re-runnable, UPSERT by id

### First run hit two issues
1. **Multi-line INSERT statements** — ~7 rows have literal newlines inside the `hours` field (`<br />\n`), my line-based extractor was cutting them off mid-statement
2. **Two different INSERT formats coexist in the bundled SQL** — most rows use the full 34-column format (newer schema with merchant fields), but ~7 rows use an older 22-column format (no merchant fields, no image_asset, no related_*_ids). Both schemas should load fine into SQLite once the multi-line issue is fixed.

### Fix
- Rewrote the extractor to accumulate lines until reaching a `;` that's NOT inside a string literal
- Added `updateStringState()` helper that tracks single-quote string boundaries with proper handling of escaped quotes (`''`)
- Re-ran. All 815 INSERTs loaded into SQLite, 814 distinct rows imported (1 was an UPSERT duplicate).

### Verification
- Total rows in `salem_narration_points`: **814** (matches bundled SQL exactly)
- Wave breakdown: W1=112, W2=85, W3=109, W4=508 (STATE.md said 817 total, off by 3 — STATE.md is slightly stale, the bundled SQL is the truth)
- 29 distinct categories (matches the rich POI taxonomy)
