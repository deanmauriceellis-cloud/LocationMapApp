### Publish pipeline fix — Room schema mismatch

**Bug found:** `publish-witch-trials-to-sqlite.js` crashed with "no such table: salem_witch_trials_newspapers" because `publish-salem-pois.js` rebuilds the entire SQLite DB from scratch (DROP + CREATE for `salem_pois` only), which destroys the witch trials tables.

**Root cause from S134:** S134 created `publish-witch-trials-to-sqlite.js` with `DELETE FROM` but no `CREATE TABLE IF NOT EXISTS`. Worked in S134 because the tables existed from a prior publish. After `publish-salem-pois.js` rebuilds the DB, they're gone.

**First fix attempt:** Added `CREATE TABLE IF NOT EXISTS` with DEFAULT values and nullable columns matching PG schema. **This caused a Room schema validation crash** — Room validates the pre-packaged DB schema against @Entity definitions on first open, and Room does NOT expect DEFAULT clauses or different NOT NULL constraints.

**Final fix:** Rewrote CREATE TABLE statements to match Room @Entity definitions exactly — no DEFAULT clauses, correct NOT NULL matching Kotlin non-null types. Schema mismatch resolved.

**Files modified:** `cache-proxy/scripts/publish-witch-trials-to-sqlite.js` — added Room-compatible CREATE TABLE IF NOT EXISTS for all 3 witch trials tables.
