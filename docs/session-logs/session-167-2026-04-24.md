### Walking Tours loading hang — root cause and fix
**Symptom:** Tour selection dialog stuck on "Loading tours…" spinner after quitting a tour.

**Root cause:** Asset DB `salem_content.db` had `PRAGMA user_version = 0`. Room `@Database(version = 9)` detected a mismatch on first launch, triggered `fallbackToDestructiveMigration()` which called `deleteDatabase()` — unlinking the file while another thread already had it open. Both `TourVM` and `SalemMainActivity` (narration points) threw `SQLiteReadOnlyDatabaseException: SQLITE_READONLY_DBMOVED`. The tour list stayed empty forever because `collectLatest` guard `if (tours.isEmpty()) return@collectLatest` never resolved.

**Fix:** Stamped `PRAGMA user_version = 9` into both:
- `app-salem/src/main/assets/salem_content.db`
- `salem-content/salem_content.db`

Room now sees version match on first launch, skips migration entirely.

**Verification:** Logcat showed `TourVM: Loaded 5 tours, 2303 POIs` with no SQLiteLog errors. `adb uninstall && adb install` performed (not `-r`) per standing protocol.

**Note for future DB bakes:** Every time `salem_content.db` is rebaked via the content pipeline, the bake script must set `PRAGMA user_version = 9` (or whatever the current Room version is) before copying to assets. If this is forgotten, the next install will trigger destructive migration again.
