## Findings before any code shipped
- `salem_pois`: 2,291 rows total, 1,821 active (deleted_at IS NULL), all have
  lat/lng. 67 rows missing `address` (60 of them active). Schema already had
  `data_source` / `confidence` / `verified_date` / `admin_dirty` columns and a
  separate (empty) `canonical_address_point_id` placeholder. No
  truth-of-record concept yet.
- `postgis_tiger_geocoder` extension lives in the `tiger` Postgres database,
  not `locationmapapp`. So the verifier needs two pg connections.
- TigerLine MA ingest is **incomplete** — `tiger.addrfeat`/`addr`/`edges`/
  `featnames` for `statefp='25'` are empty (per OMEN tigerline session-001/002
  reports the import was still running). `tiger.geocode()` returns 0 rows for
  every real Salem address tested ("115 North St / 32 Derby Square / 86
  Lafayette St"). The 6087 ms cost on the second probe was `geocode()` burning
  through fuzzy fallback retries against an empty dataset.
- `massgis` schema **is** loaded (193 tables) including `l3_parcels_essex`
  (parcel polygons with GIST index, MultiPolygon@4269) and
  `basestreetswithlabels`. **But** the L3 product as loaded does NOT include
  the joined assessor table — so `l3_parcels_essex` has only `objectid`,
  `loc_id`, `town_id`, `site` and no parsed address fields. No `addr_num` /
  `full_str` / `city` to look up against.
- Net: **no usable address-to-coordinate ground-truth source on this machine
  right now.** Verifier work is blocked. Operator confirmed they know
  TigerLine is still populating.

## Pivot: ship schema + UI scaffolding now, defer the verifier query

### Schema migration applied
- File: `cache-proxy/scripts/poi-verify-2026-04-23/01-add-location-verify-columns.sql`
- Adds 7 nullable + defaulted columns to `salem_pois`:
  `lat_proposed`, `lng_proposed`, `location_source`,
  `location_status` (default `'unverified'`),
  `location_truth_of_record` (default `false`),
  `location_drift_m`, `location_geocoder_rating`, `location_verified_at`.
- Adds 3 partial indexes:
  `idx_spois_location_status` (where deleted_at IS NULL),

- Backfill: 60 active rows with no `address` flipped to
  `location_status = 'no_address'`.
- Idempotent (`IF NOT EXISTS` everywhere). `salem-schema.sql` not touched —
  it's an unreferenced reference doc that has already drifted from live.

### Backend: cache-proxy admin API (`cache-proxy/lib/admin-pois.js`)
- Whitelisted `location_truth_of_record` + `location_status` for PUT
  `/admin/salem/pois/:id`. Verifier-owned columns
  (`lat_proposed/lng_proposed/location_source/location_drift_m/...`) are
  intentionally NOT whitelisted — admin must use the new endpoint to commit.
- New endpoint: `POST /admin/salem/pois/:id/accept-proposed-location` —
  copies `lat_proposed → lat`, `lng_proposed → lng`, sets
  `location_status='accepted'`, clears the proposal columns, stamps
  `admin_dirty`. 409 if no proposal exists. Mirrors `/move` semantics.
- Node syntax-checked.
