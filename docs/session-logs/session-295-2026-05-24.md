### S295 — `ContentManifestVerifier` shipped (plan S3 steps 1–4 ✅; step 5 carried)

**NEW `core/.../activation/ContentManifestVerifier.kt`:**
- Pubkey load: `KeyFactory("RSA")` + `X509EncodedKeySpec` over the SPKI DER.
- `verifyCheap()` (every launch): verify `.sig` over the verbatim manifest bytes → parse → recompute canonical `manifestHash` (self-consistency, catches a stale/edited hash field) → recompute each in-scope file SHA-256 from disk bytes (streamed via `DigestInputStream`) and compare.
- `verifyFull()` (activation only): + bulk-asset presence/size via an injectable `largeAssetStat`; tile DB (asset pack) stays Phase-2-deferred → `largeVerified=false` but still passes.
- Returns `Result.Ok(manifestHash, roomVersion, roomIdentityHash, largeVerified)` / `Result.Fail(reason)`; `manifestHash` is the handshake input (`SHA256(nonce‖manifestHash)`).
- **JVM-testable by construction:** all inputs injected (pubkey bytes + `openAsset` lambda + optional `log`); no `Context`/`AssetManager`/`android.util.Log` on the verified path (avoids needing Robolectric). Production wiring = companion `forAndroidAssets(context, R.raw.content_manifest_pubkey)` — app passes the resId so `:core` stays decoupled from `:app-salem`'s R.
