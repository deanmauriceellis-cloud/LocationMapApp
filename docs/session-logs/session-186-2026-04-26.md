### Mid-session reframings (load-bearing — do not lose)

Operator clarified the design twice during the session, each time tightening the model:

1. **Mode-dependent Layers prefs.** Initial implementation had ONE pref per Layers checkbox. Operator: explore mode (no tour) defaults to BOTH checkboxes ON (everything visible + audible by default — "Explore Salem"); tour mode defaults to BOTH OFF (more restrictive — only is_tour_poi narrates). Each mode remembers its own pref independently. Implemented via PREF_POI_HIST_LANDMARK / PREF_POI_HIST_LANDMARK_TOUR pair (same for civic). The Layers menu reads `MenuEventListener.isTourActive()` to pick which pref to render and write.

2. **`is_tour_poi=true` is the force-visible AND force-audible override.** Initial filter gated EVERY POI through the Layers checkboxes; First Church Meetinghouse 1692 Site (is_tour_poi=true + HISTORICAL_BUILDINGS) was hidden when the Hist Landmark checkbox was OFF. Operator rule: the Layers checkboxes only add/remove the *additional* class they unsilence; the tour-flagged baseline always shows + narrates. Fixed by short-circuiting the marker filter on `if (p.isTourPoi) return@filter true` (mirrors the runtime narration gate's `if (point.isTourPoi) return true` in NarrationGeofenceManager.isTourEligible).

3. **1-hour dedup must hold across walk-sim restarts.** Operator: "until next cold start, if we narrated anything, it should not be repeated for an hour." Walk-sim start at SalemMainActivity:1660 was unconditionally calling `narrationGeofenceManager.resetSession()` (wiping narratedAt + cooldowns + persistent SharedPreferences mirror), so each walk-sim restart re-armed the same POIs. Removed that call. Dedup now persists across walk-sim start/stop, app pause/resume, and process death (S125 SharedPreferences mirror); only cleared by a true cold start (onCreate with savedInstanceState=null at line 674).
