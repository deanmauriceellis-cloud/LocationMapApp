### Generator overhaul (`generate-historical-narrations.js`)

- 100–250 word HARD CAP (was up to 500). Validator truncates over-cap
  output at the last sentence boundary; rejects unrecoverable single-
  sentence overruns.
- STRICT_RULE 13: anti-repetition / anti-padding — forbids restated
  facts, closure flourishes, redundant style-marker repetition. Each
  sentence must carry a NEW grounded fact.
- STRICT_RULE 14: no meta-narration ("you are looking at…", "imagine
  yourself", reader-address phrasings).
- STRICT_RULE 15: name anchor in the first 240 chars. Caught a
  "Fairbanks House" hallucination on Witch House. Forces retry up to 3
  attempts.

  voice; post-1860 subjects → present-day Salem voice. Tribute-mode

## Audit log infrastructure (operator-requested mid-session)

Operator: "I need audit control over every manual change made so I can
revert it. Every event change in the Web Admin tool auditable and
reversible. Including what we do in automation."

### Schema (`scripts/install-audit-trigger.sql`)

- New `salem_audit_log` table — `id, recorded_at, table_name, entity_id,
  action, changed_fields[], old_values JSONB, new_values JSONB, actor,
  source, reverted, reverted_at, reverted_by, revert_audit_id`.
- PG row-level trigger `salem_audit_trigger_fn()` AFTER
  INSERT/UPDATE/DELETE. Skips meta-only changes (admin_dirty,
  admin_dirty_at, updated_at). Reads `app.actor` + `app.source` via
  `current_setting('app.actor', true)` — caller sets these via
  `SET LOCAL "app.actor" = '...'`.
- Trigger attached to: salem_pois, salem_tours, salem_tour_legs,
  salem_tour_stops, salem_poi_categories, salem_poi_subcategories,
  salem_witch_trials_articles/newspapers/npc_bios, salem_geocode_blacklist.

### Cache-proxy endpoints (`lib/admin-audit.js`)

- `GET /admin/salem/audit` — paginated list, filterable.
- `GET /admin/salem/audit/stats` — summary counts.
- `GET /admin/salem/audit/:id` — full row with old/new JSON.
- `GET /admin/salem/audit/entity/:table/:id` — per-entity timeline.
- `POST /admin/salem/audit/:id/revert` — applies inverse op (UPDATE
  back, DELETE the inserted row, INSERT back the deleted row), marks
  the original audit row reverted, writes a new audit row capturing
  the revert.

### Web admin Audit tab (`web/src/admin/AuditTab.tsx`)

New "Audit" header toggle next to POIs/Tours/Witch Trials/Lint/Geocodes.
Stats strip + filter bar + timeline table with action color-coding,
relative time, expand-to-diff (before/after per field), per-row
Revert button with reason prompt. Auto-refresh every 15s.
Pagination.

E2e revert test verified: lat-shift on lappin_park → POST revert →
