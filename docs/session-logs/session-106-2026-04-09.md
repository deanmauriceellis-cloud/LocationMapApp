### Decisions

- **Direct browser → Oracle calls** (no Vite proxy): permissive CORS makes it work end-to-end without the cache-proxy round-trip. Avoids hardcoding the proxy rule and matches the spec.
- **Configurable base URL via Vite env var** `VITE_SALEM_ORACLE_URL`: operator can override at dev-server start time if running the admin tool from a LAN host instead of the dev box. Default `http://localhost:8088`.
- **Status polling at mount + every 30s** (`ORACLE_POLL_MS=30_000`): cheap (sub-millisecond on Oracle side), short timeout (5s) so the pill flips fast when the testapp bounces. Click pill to force immediate re-poll. State machine: loading → ready / unavailable.
- **Three pill states**:
  - `loading` — grey "Oracle: …" while first poll in flight
  - `ready` — emerald "Oracle: ready" with rich tooltip showing fact/PS/POI/newspaper counts + history turn count
  - `unavailable` — rose "Oracle: down" with tooltip pointing at `bash ~/Development/Salem/scripts/start-testapp.sh`
- **Sub-dialog as nested `<Dialog>`** at z-1100 (main edit dialog is z-1000). Headless UI handles nesting cleanly; both dialogs can be open at once visually.
- **Reset checkbox auto-flips OFF after first generation**: spec says first call should reset history (cross-POI bleed protection) but iterate calls should not. Auto-flipping makes the iterate workflow feel natural without the operator having to remember to uncheck it.
- **Insert buttons filtered through `has(field)`**: tour POIs without `short_narration` / `long_narration` simply don't see those buttons. The General-tab description button is just the same launcher in compact form — both buttons open the same sub-dialog state because the master plan says §1364 "Add the same button to the General tab."
- **Audit log on INSERT, not on GENERATE**: generate-but-discard is just thinking out loud and shouldn't pollute the log. Insert is the moment of acceptance. Stored in `localStorage[salem-oracle-audit]`, capped at 500 entries FIFO, best-effort (quota errors are warned to console, not surfaced).
- **Concurrent call serialization** at the client (per master plan §1383): the `runOracleAsk` function early-returns if `oracleBusy` is true. Belt-and-braces with Ollama's natural serialization on the GPU.
- **Spinner UX**: violet panel with text "Asking the Oracle… typically 5-15s. The LLM is gemma3:27b on the workstation GPU; concurrent calls would queue, so this is sequential." Sets expectations without artificial min-display delay.
- **Replace, not append**: insert buttons replace the field via `setValue(field, text, { shouldDirty: true, shouldTouch: true })`. Operator is committing intentionally after seeing the result.
