### Decisions

- **One dialog component** (`PoiEditDialog.tsx`), kind-aware via prop. Common fields render unconditionally; per-kind fields are gated by `kind === 'tour'/'business'/'narration'` checks. Master plan tab structure: General · Location · Hours & Contact · Narration · Provenance · Linked Historical Content · Danger Zone.
- **react-hook-form for dirty tracking** — `formState: { dirtyFields }` drives the PUT payload; only fields the user actually touched are sent. Avoids accidentally clobbering server-only fields and aligns with the partial-update endpoint's intent.
- **JSONB fields** rendered as `<textarea>` with `JSON.stringify(value, null, 2)` defaults. On submit: `JSON.parse` each dirty JSONB field; on parse failure, surface inline error and abort save. Empty textarea → `[]`.
- **Dropdowns:** category/business_type uses a free-text input + `<datalist>` of observed values from `knownCategories` (computed in AdminLayout from `byKind`). New values allowed.
- **Soft delete** in Danger Zone tab: button → confirm sub-dialog → DELETE; on success, AdminLayout patches the deleted row with `deleted_at = NOW()` so the tree immediately ghosts it (with show-deleted toggle off, it disappears).
- **Open trigger:** marker click in `AdminMap` (per master plan §1296). Tree click selects + flies to but does NOT open the dialog — this avoids launching a modal whenever the operator is just navigating the tree.
- **9P.10a stub:** Linked Historical Content tab renders "No links yet — see Phase 9Q (Salem Domain Content Bridge)" as a placeholder. Actual implementation lands in a later step.
- **9P.10b placeholder:** No Oracle button in this step. Comment in `Narration` tab marks where the button will land in 9P.10b.

**Save flow:**
- `handleSubmit(onSubmit)` → `buildPayload(values, dirtyFields)` → only the dirty field set is sent
- JSONB fields: `JSON.parse(textareaValue)` per dirty field; on parse failure, return `{ error: 'JSON parse error in field "X": ...' }` and abort
- Numeric fields: empty string → null, otherwise `parseFloat` and `Number.isFinite` check; non-numeric aborts
- Boolean: `!!raw`
- Date: empty string → null, otherwise pass YYYY-MM-DD through
- Default: empty string → null
- POST `PUT /api/admin/salem/pois/:kind/:id` with the partial payload
- 200 response → `onSaved(kind, updated)` → AdminLayout patches `byKind` row → `reset(buildDefaults(updated, kind))` so dirty tracking restarts from saved baseline → `onClose()`
- Non-2xx → keep dialog open, set `saveError` state, form state preserved
- Empty payload (somehow no dirty fields after coercion) → close without API call

**Cancel flow:**
- `handleClose` → if `isDirty`, `window.confirm("You have unsaved changes. Discard and close?")`; if false, abort
- Otherwise reset error states and call `onClose`

**Soft delete flow:**
- `confirmDelete` state controls the inline confirm prompt (no nested Headless UI dialog — keeps the focus stack simple)
- DELETE request with no body
- Success → `onDeleted(kind, id, body.deleted_at)` → AdminLayout patches the row's `deleted_at` in `byKind` → tree ghosts the row, map drops it
- Failure → `deleteError` state, retry available

**Footer save bar:**
- Left: dirty field count "N field(s) modified" or "No changes" (or `saveError` text in red on error)
- Right: Cancel + Save buttons; Save disabled when `!isDirty || isSubmitting`

**Open trigger (per master plan §1296):**
- AdminLayout `handleTreeSelect` → sets `selectedPoi`, does NOT open dialog (operator may be browsing)
- AdminLayout `handleMapSelect` → sets `selectedPoi` AND `editOpen=true` (operator clicked a marker, so they want to edit)
