### ‚Äî SalemIntelligence narration sync

**Goal:** Pull refined narrative text from SalemIntelligence (:8089) into all linked POIs in `salem_pois`.

**Script created:** `cache-proxy/scripts/sync-narrations-from-intel.js`
- Fetches from two SalemIntelligence sources per entity:
  1. `GET /api/intel/entity/{id}/narration` ‚Äî LLM-generated narrations (short, medium, long, entity)
  2. `GET /api/intel/poi-export` ‚Äî BCS extracted descriptions (short_description, long_description, origin_story)
- Field mapping: short_narration‚Üíshort_narration, long_narration‚Üílong_narration, medium_narration‚Üídescription, entity_narration‚Üínarration_pass_2, BCS short_description‚Üíshort_description, BCS origin_story‚Üíorigin_story, BCS long_description‚Üídescription (fallback)
- Overwrite mode (designed for repeated refresh)
- Direct PG updates in single transaction, concurrency 10 for API calls
- Supports `--dry-run` and `--concurrency N` flags

### ‚Äî Drop multipass narration (pass_2, pass_3)

**Decision:** Multipass narration rotation (pass_2/pass_3 columns) was dead weight. pass_3 had 12 entries out of 2,190 POIs. pass_2 duplicated what long_narration provides. Visit-count gated TTS rotation was invisible to users.

**Data migration:**
- 717 POIs had pass_2 content but no long_narration ‚Üí promoted pass_2 into long_narration
- long_narration coverage: 839 ‚Üí **1,556** POIs (+717)
- Columns `narration_pass_2` and `narration_pass_3` dropped from PG

**Simplified narration model:**
- `short_narration` ‚Äî geofence walk-by TTS announcement (~20-50 words)
- `long_narration` ‚Äî detail view / linger content (~200-500 words)

**Files changed:**

### ‚Äî Room schema crash fix (narration_points + salem_pois)

After dropping pass columns from Kotlin entities, the bundled SQLite still had the old columns in both `narration_points` and `salem_pois` tables. Room schema validation rejected the mismatch on startup ‚Üí FATAL crash.

**Fix:** Recreated `narration_points` table without pass columns via SQLite table rebuild. Dropped and recreated `salem_pois` table (publish script's `CREATE TABLE IF NOT EXISTS` was preserving old schema). Both tables now match their Room entity definitions.

### ‚Äî POI detail sheet: tap-to-speak sections + TTS reorder

**User request:** Tapping Overview/About/The Story sections should interrupt current narration and play that section. Address should be tappable too. TTS should start at Overview, not address.

**Changes to `PoiDetailSheet.kt`:**
- Overview, About, The Story labels AND body text are now clickable ‚Äî tap interrupts all TTS (`stopNarration()`) and plays that section
- Address text is clickable ‚Äî tap reads "Located at [address]"
- TTS auto-play reordered: name+category intro ‚Üí Overview ‚Üí About ‚Üí Story (address/phone/website removed from auto-play, available on tap)
- Added `interruptAndSpeak()` helper method

### ‚Äî Missing POI circle icons

SalemPoi unified categories (`parks_rec`, `food_drink`, `entertainment`, etc.) had no entries in `MarkerIconHelper.CIRCLE_ICON_MAP`. These POIs fell through to a 5dp dot fallback ‚Äî barely visible on the map.

**Fix:** Added 19 default `CIRCLE_ICON_MAP` entries for all SalemPoi coarse categories, mapping to appropriate default icons in each asset folder (e.g. `parks_rec` ‚Üí `parks_rec/park.png`, `food_drink` ‚Üí `food_drink/restaurant.png`).

### ‚Äî Zoom-out marker sticky size bug

`refreshNarrationIcons()` only updated markers inside the viewport bounding box. Edge-of-screen markers kept their large zoom-18 icons when zooming out.

**Fix:** Now refreshes ALL markers on zoom bucket change (icons are cached, so per-marker cost is just an assignment). Removed viewport-only optimization.

### ‚Äî Dwell expansion radius tightened

Dwell reach-out system was expanding to 100m, narrating distant mundane POIs (parking lots, real estate offices at 77m). Logs showed: `20m ‚Üí 35m ‚Üí 60m ‚Üí 100m`.

**Fix:** Tightened ladder to **20m ‚Üí 35m ‚Üí 50m max**. Removed `DWELL_RADIUS_WIDE_M` (60m) step. Reduced max from 100m to 50m.
