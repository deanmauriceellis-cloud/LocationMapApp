## P2 + P3 done — router live in Salem app

### `:core` router (P2 + P2b)

- `core/src/main/java/com/example/locationmapapp/core/routing/`:
  - `RouteTypes.kt` — `RoutingLatLng`, `NodeRef`, `RouteEdgeStep`, `RouteResult`.
  - `RoutingBundle.kt` — immutable in-memory CSR (compressed sparse row) graph. Each upstream edge is stored once in the edge tables and twice in the adjacency arrays (per direction) with a `reversed` flag so polylines can be emitted in the direction of travel. Spatial KNN via a fixed-cell grid (~165m cells).
  - `RoutingBundleLoader.kt` — opens the SQLite read-only via Android `SQLiteDatabase`, refuses unknown `schema_version`, builds the CSR.
  - `Router.kt` — pure-Kotlin Dijkstra with binary heap, planar SRID-4269 KNN to match TigerLine's `<->` semantic, undirected pedestrian graph. Public API: `nearestWalkableNode`, `route(from→to)`, `routeMulti(stops)`. Returns `RouteResult` with geometry + per-edge breakdown for turn-by-turn.
  - `TurnByTurn.kt` — `synthesize(RouteResult) → List<TurnStep>`. Groups consecutive same-`fullname` edges, classifies bearing changes (continue / slight / turn / sharp / U-turn), formats with imperial distance.
