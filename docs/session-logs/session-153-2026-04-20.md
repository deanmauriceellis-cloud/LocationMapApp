## Demo crash — 2026-04-20 12:53:25 (post-Webex debrief)

**Operator report:** "it crashed when I tried to demo it" (post-counsel/Webex return).

**Root cause (from crash buffer):**
```
04-20 12:53:25 FATAL EXCEPTION: main
android.database.sqlite.SQLiteException: no such table: salem_witch_trials_npc_bios
  (code 1 SQLITE_ERROR): SELECT * FROM salem_witch_trials_npc_bios ORDER BY tier ASC, name ASC
  at WitchTrialsNpcBioDao_Impl$5.call(WitchTrialsNpcBioDao_Impl.java:170)
```
Structurally identical to the S150 `salem_witch_trials_newspapers` missing-table crash — the demo opened the Witch Trials **People** panel and Room hit an unbaked table.

**Scope:** diffing `salem_content.db .tables` against `@Entity` declarations shows **TWO** tables missing (not just the one that crashed):
- `salem_witch_trials_npc_bios` — 49 rows in PG (the People panel crash)
- `salem_witch_trials_articles` — 16 rows in PG (History 4×4 tiles — next crash waiting)

Both are missing from **both** the source DB (`salem-content/salem_content.db`) and the APK assets DB. `salem_witch_trials_newspapers` is present — that was fixed in S150.

**Why:** the two bundle scripts (`bundle-witch-trials-npc-bios-into-db.js`, `bundle-witch-trials-into-db.js`) write only to `ASSETS_DB`. The S152 SI KB absorption ran `publish-salem-pois.js`, which rebuilds the source DB and copies it over the assets DB — silently wiping both tables because the source DB never had them. The S150 newspaper fix was: write to source DB, `CREATE TABLE IF NOT EXISTS` to survive source-DB regens, mirror source → assets. The npc_bios + articles scripts never got that same treatment.

### Fix applied

Patched both bundle scripts to match the S150 newspaper pattern:

- `cache-proxy/scripts/bundle-witch-trials-npc-bios-into-db.js` — writes to `salem-content/salem_content.db`, adds `CREATE TABLE IF NOT EXISTS salem_witch_trials_npc_bios`, mirrors source → assets at end.
- `cache-proxy/scripts/bundle-witch-trials-into-db.js` — same treatment for `salem_witch_trials_articles`.

Ran both. Post-run verification (both DBs identical):

```

salem_witch_trials_npc_bios     = 49 rows
room_master_table identity_hash = 458bb11df51a54f5284a03ef1d2913aa  (v8, matches Room expectation)
```
