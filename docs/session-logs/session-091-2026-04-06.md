## Walk Simulator Bug Fixes

**User report:** Tour 1 emulated walk: not getting all non-paid POIs, walk gets stuck, clicking on map doesn't recover from emulated position.

**Root cause analysis — 3 bugs found:**

### Bug 1: Walk sim doesn't stop on user interaction
- **File:** `SalemMainActivity.kt`
- Long-press handler calls `viewModel.setManualLocation(p)` but walk sim keeps running in background coroutine
- Walk sim overwrites user's position every 1 second, snapping back to the route
- Single-tap also didn't stop the walk sim
- **Fix:** Added `if (walkSimRunning) stopWalkSim()` to both single-tap (line ~641) and long-press (line ~668) handlers

### Bug 2: interpolateWalkRoute() residual distance bug
- **File:** `SalemMainActivity.kt:interpolateWalkRoute()`
- When carry-over distance from previous segment exceeds current segment length, the while loop is never entered
- `residualM` was never reduced by the short segment's distance, causing it to accumulate incorrectly
- Could cause walk to stall or bunch points incorrectly in areas with many short route segments
- **Fix:** Added early check: `if (residualM >= segDist) { residualM -= segDist; continue }`

### Bug 3: narratedToday persists across walk sim runs
- POIs narrated in earlier walk sim runs (or earlier testing) are marked in `narratedToday` set
- Won't trigger again until midnight reset
- Walk sim starting a new run should be a fresh experience
- **Fix:** Added `narrationGeofenceManager?.resetSession()` at the start of `startWalkSim()`
