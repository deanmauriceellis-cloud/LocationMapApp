### Polygon library + animation overlays
- Operator: "we need to create our own library of polygons that we will animate, we will do more than just water." Saved as design direction.
- Operator: "we don't need OSM! We have tigerline and massgis — put that in memory!" Saved `feedback_no_osm_use_local_geo.md` HARD RULE: spatial data comes from local TigerLine + MassGIS Postgres ingests, never OSM PBFs / water-polygons-split shapefiles. Pinned in MEMORY.md.
- Built extract pipeline:
  - `tools/wickedmap-polygons/extract-water-from-tiger.py` — queries `tiger.areawater` for Salem bbox, emits GeoJSON FeatureCollection with `kind="water"`, `subkind` (mtfcc → pond/bay/river/...), name. Result: 545 water features (411 KB).
  - `tools/wickedmap-polygons/extract-cemeteries-from-massgis.py` — queries `massgis.openspace WHERE prim_purp='H' AND name~/cemetery|burying|.../`, emits `kind="cemetery"`. Result: 97 cemeteries (preserved 12-pt shape detail after `ST_SimplifyPreserveTopology(geom, 0.00003)` fix; original 5.0° tolerance had collapsed every cemetery to a 4-point quad).
- Built `PolygonLibrary.kt` — loads `assets/wickedmap/polygons.json` at app start, parses GeoJSON, indexes by `kind`. Loads 642 polygons in ~1 s.
- Animation overlays:
  - `AnimatedWaterOverlay.kt` — whitecaps. Iterated several times: started with hardcoded 4-water-body polygon list, replaced with PolygonLibrary lookup. Scaled density per polygon. Added per-polygon-bbox visibility culling (drop fps from 16 → 27 with full library loaded).
  - `FireflyOverlay.kt` (originally `CemeteryWispOverlay`, renamed for kind-agnostic reuse) — spectral ghost orbs in cemeteries. Tuned twice: first iteration was too sparse + too dim + used `BlurMaskFilter` which silently dropped on this SurfaceView; second iteration dense bright fireflies with sharp 8th-power blink; final iteration slow ghostly fade-in/drift/fade-out at 4.5-7.5 s per orb with `sin²` intensity, spectral cold-green color.
