### Orientation finding (V1 ship-blocker)

Operator reported app stuck in landscape on Pixel 8 with different screen behavior vs Lenovo. Diagnosed:
- `SalemMainActivity` has **no `android:screenOrientation` lock** in `app-salem/src/main/AndroidManifest.xml` (line 61-66); follows system rotation. No programmatic `setRequestedOrientation` calls anywhere in `app-salem/src/`.
- Pixel system state: `accelerometer_rotation=0` (auto-rotate OFF) + `mCurrentRotation=ROTATION_90` — system was manually rotation-locked to landscape, so the app inherited landscape.
- More important finding: **the app has no `layout-port/` or `layout-land/` resource qualifiers** — single `layout/` directory designed implicitly around Lenovo (natural landscape). On a Pixel 8 portrait the existing landscape-geometry layouts will squish/clip.

Operator decision: **full multi-form-factor V1 support — both portrait + landscape on phones AND tablets.** Stated rationale: *"that is why I wanted to get the Pixel — to surface these issues."* + *"We need to support ALL android devices in both orientations — I will probably need to get a few more to test with but we start with these 2."* Lock-landscape shortcut declined.
