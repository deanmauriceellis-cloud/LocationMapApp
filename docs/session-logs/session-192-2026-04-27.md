## Major task — Historical Narration system (new)

Operator: build a third narration type alongside `short_narration` /
`long_narration`. **Every HISTORICAL_BUILDINGS POI older than 1860 gets
its own historical narration**, grounded only in pre-1860 facts. No
present-day or post-1860 references. Empty if facts insufficient.
Powers a future "Historical Tours" mode that doesn't mix with
short/long narration.

### Operator design rules (locked in dialog)

1. **Year cutoff**: < 1860. Use BCS-derived year if LMA's
   `year_established` is NULL or wrong (Option B).
2. **Length**: rich pre-1800 content where grounded — up to 500 words.
   Standard-length (60-150 words) for 1800-1859. EMPTY if insufficient.
3. **Highly templated and prompt-engineered**, with SalemIntelligence as
   the grounded fact source.
4. **LMA is system of record**; SalemIntelligence is content-generation
   source only. One-way: SI → LMA.
5. **Probe for original owner / builder / architect** as part of the
   prompt template.
6. **PG-13 / IARC Teen** content rule applies (already a memory).
7. Statues / monuments / plaques / memorials → NOT eligible regardless
   of LMA year (post-1860 artifacts even when commemorating earlier
   subjects).
8. **Temporal framing** — use vocabulary appropriate to entity's era:
   - 1626-1691: "Massachusetts Bay Colony", "the colony", "colonial
     settlement"; for 1626-1660 may reference "Old Planters of Salem"
     (Conant's founding group) when grounded
   - 1692-1775: "Province of Massachusetts Bay", "the colony"
   - 1776-1779: "the colony" or "the newly-declared United States"
   - 1780+: "Commonwealth of Massachusetts", "United States"
   - Pre-1776: ban "United States", "America", "American"
   - Pre-1780: ban "Commonwealth of Massachusetts"

### Generator script — `cache-proxy/scripts/generate-historical-narrations.js`

Pipeline per POI:
1. GET `/api/intel/entity/{intel_entity_id}/dump` → ground truth
2. **Pre-eligibility filters** (in order):
   - Skip if name matches `/statue|monument|plaque|marker|memorial|cenotaph|obelisk|tablet|bust/i`
   - effective_year = `mode_c.built_year` ?? `mode_c.construction_start_year` ?? `lma.year_established`
   - Skip if effective_year >= 1860 OR null
3. Witch-trial signal: any relation/description matching
   `witch.*trial|corwin|hathorne|parris|proctor|samuel sewall|cotton mather|tituba|bridget bishop|rebecca nurse|giles corey`
4. Length bucket: `RICH` (pre-1800) or `STANDARD` (1800-1859)
5. Build GROUND_TRUTH block from structured fields + pre-1860
   description sentences + filtered relations (skip neighbor_of, skip
   anything with year >= 1860)
6. Pre-strip GROUND_TRUTH lines containing `historic district`,
   `national register`, `national historic landmark`, year >= 1860,
   museum/visitors/admission tokens
7. POST `/api/intel/chat` with templated prompt. Two attempts on
   failure.
8. Post-process answer:
   - Strip gemma3 special tokens (`<unused...>`, `<|...|>`)
   - Strip leading non-ASCII glyphs
   - Anchor to first occurrence of POI name within first 250 chars
9. Validate:
   - Year regex `>= 1860` reject
   - Banned phrases (50+ patterns including spelled-out centuries,
     modern continuity paraphrases, NRHP/historic-district designations,
     museum-operation language)
   - Pre-1776 banned: `united states`, `america`, `american`
   - Pre-1780 banned: `commonwealth of massachusetts`
   - ASCII ratio < 0.95 reject (catches CJK gemma flakes)
   - Word count floor: RICH >= 200, STANDARD >= 60
10. UPDATE `salem_pois SET historical_narration = ...` (skipped on
    `--dry-run`).

CLI: `--dry-run | --force | --verbose | --limit=N | --ids=a,b,c`.
Default behavior: skip POIs that already have `historical_narration`
populated, so re-running is safe and only fills gaps.
