## Key architectural decision: V1 is fully offline

Operator direction: "For the vision of the version that will get released as our commercial version 1 - it will not require internet ability for anything. I don't plan to host a LLM/KB for the salem tour our first release. I want it all self contained."

**Decision:** SalemIntelligence is a build-time data source only for v1. We harvest BCS data at dev time → PG → salem-content pipeline → Room DB → APK. No runtime API calls, no hosted service dependency.

**Implications:**
- Only `GET /api/intel/poi-export` matters — called once at dev time
- Tiers 2-4 (context, nearby, chat) are post-v1
- "Where's a dentist?" solved by offline layer toggles (HEALTHCARE defaultEnabled=false)
- Store `entity_id` FK anyway for v2 online features (zero cost)
- No salemIntelClient.ts, no status pills, no runtime SalemIntelligence dependency in v1

Memory saved: `project_v1_offline_only.md`
