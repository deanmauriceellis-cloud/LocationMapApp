## Phase 9A+ Step 1: Offline Map Tiles

### Research
- **Current tile setup:** Three online sources (Esri satellite, OSM Mapnik, CartoDB Dark) via `TileSourceManager.kt`. No offline support. `osmdroid-mapsforge:6.1.18` dependency exists but unused.
- **osmdroid offline mechanism:** `MapTileFileArchiveProvider` auto-discovers `.sqlite`/`.zip`/`.mbtiles`/`.gemf` files in `osmdroidBasePath`. `DatabaseFileArchive` reads tiles from SQLite with schema: `tiles(key INTEGER PRIMARY KEY, provider TEXT, tile BLOB)`. Key = `MapTileIndex.getTileIndex(zoom, x, y)` using variable-width bit packing.
- **Bounding box:** Tour stops span 42.5126-42.5234 lat, -70.8964 to -70.8827 lon. With buffer for panning: 42.508-42.530 lat, -70.905 to -70.876 lon (~2.5km × 2.4km).
- **Tile count:** 696 tiles/source × 2 sources (Esri + OSM) = 1,392 tiles at zoom 14-18.
- **Scoped storage:** osmdroid defaults to `Environment.getExternalStorageDirectory()` which may not be writable on Android 10+. Must use `context.getExternalFilesDir()`. Solution: set `osmdroid.basePath` in SharedPreferences before `Configuration.load()`.

### Implementation
1. **Created `tools/download_salem_tiles.py`** — Downloads tiles for Salem bbox (zoom 14-18), packages as osmdroid-compatible SQLite archive. Esri satellite + OSM Mapnik. Rate-limited at 150ms/request.
2. **Created `OfflineTileManager.kt`** — Two methods:
   - `configureStoragePath()` — Sets osmdroid base path in SharedPreferences (scoped storage compat)
   - `extractArchiveIfNeeded()` — Copies bundled `.sqlite` from APK assets to osmdroid base path on first launch. Versioned (CURRENT_VERSION=1) to support future tile updates.
3. **Updated `WickedSalemApp.kt`** — Three-step init: (1) configure storage path, (2) load osmdroid config, (3) extract tiles. Ordering ensures base path is in prefs before load() reads it.
4. **Build verified** — `compileDebugKotlin` passes with new code.
5. **Tile download running** — ~3.5 minutes, output to `app-salem/src/main/assets/salem_tiles.sqlite`.

### Key decisions
- Used `DatabaseFileArchive` format (osmdroid's native SQLite) — auto-discovered, no custom provider needed
- Satellite + Street tiles only (no Dark mode offline — acceptable for MVP)
- Zoom 14-18 covers walking tour needs (14-15 overview, 16-17 street level, 18 detail)
- Archive is separate from tile cache — not subject to cache eviction
- Version number in SharedPreferences allows updating bundled tiles in future app releases

### Bug fix: composite primary key
First download run had `key INTEGER PRIMARY KEY` — single column PK. Mapnik tiles with same z/x/y overwrote Esri tiles (Esri count dropped from 696 → 192). Fixed to `PRIMARY KEY (key, provider)` matching osmdroid's SqlTileWriter schema. Re-downloaded successfully.

### Download results
- **1,392 tiles** (696 Esri satellite + 696 OSM street), **0 errors**
- **Database size:** 18.0 MB (`app-salem/src/main/assets/salem_tiles.sqlite`)
- **APK size:** 30 MB (was 12 MB, +18 MB tiles) — well within Play Store 150 MB limit
- **Download time:** 321 seconds (~5.4 minutes)
- **Build:** `assembleDebug` passes

### Files created/modified
