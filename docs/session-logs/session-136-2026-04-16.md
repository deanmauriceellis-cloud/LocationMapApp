### Discovered: dual walk-sim bug

When `/walk-tour?startTour=true` is called via the debug HTTP endpoint, it starts the HTTP walk AND activates the tour engine. The tour activation can trigger a second UI walk-sim if the walk button is tapped on the device. Two concurrent GPS emitters cause 180° bearing oscillation. Pre-existing issue, not introduced this session. Workaround: don't tap the walk button when the HTTP walk is running.
