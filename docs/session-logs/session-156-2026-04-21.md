- Baked `salem_content.db` `room_master_table.identity_hash = 458bb11df51a54f5284a03ef1d2913aa`.

### Install protocol learning

`adb install -r` (`-r` = replace) keeps app data but SIGKILLs the running process. When combined with an asset-DB refresh, that kill can leave Room's SQLite in a stale/unrecoverable state on the next launch. For **any session that rebakes the bundled `salem_content.db`**, prefer `adb uninstall && adb install` over `install -r`. This guarantees a fresh asset copy and a clean Room open.
