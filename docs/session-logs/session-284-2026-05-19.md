### Root cause

In `sheet_collection.xml` (S275 file from the Katrina's Collection Phase 2/3 tabbed-UI ship):

- Lines 115-153 wrap `tabList` (TextView @ line 127) and `tabBadges` (TextView @ line 140) inside a `collectionTabs` LinearLayout.
- Lines 156-163 declare `tabIndicator` as a direct child of the outer ConstraintLayout, constrained to `@id/tabList` start/end.
- `tabList` is NOT a sibling of `tabIndicator` — it's nested one level deeper.
- ConstraintLayout only allows constraint references between **direct siblings**. Lint correctly flagged both lines 162 and 163.

Read `CollectionSheet.kt:460-477`:

```kotlin
private fun showTab(showBadges: Boolean) {
    listView.visibility = if (showBadges) View.GONE else View.VISIBLE
    gridView.visibility = if (showBadges) View.VISIBLE else View.GONE
    tabList.setTextColor(...)
    tabBadges.setTextColor(...)
    val parent = tabIndicator.parent as? androidx.constraintlayout.widget.ConstraintLayout
    parent?.let {
        val anchor = if (showBadges) tabBadges else tabList
        (tabIndicator.layoutParams as androidx.constraintlayout.widget.ConstraintLayout.LayoutParams).apply {
            startToStart = anchor.id
            endToEnd = anchor.id
            tabIndicator.layoutParams = this
        }
    }
}
```

Same bug at runtime — `startToStart = anchor.id` where anchor is a grandchild. So the "sliding amber underline under active tab" effect from S275 had been silently broken on device too since the day it shipped. Indicator just sat at parent default (top-start with width 0dp = invisible). Nobody noticed because it's a 2dp underline and the tab-text-color flip is the more obvious active-tab signal.

### Fix decision

`AskUserQuestion` round 1 — operator picked **Proper layout refactor (Recommended)** over surgical XML-only fix or baseline suppression.

### Refactor

`grep -rn "collectionTabs"` confirmed only the XML referenced it (Kotlin doesn't lookup or bind to it). Safe to remove the wrapper.

Edited `app-salem/src/main/res/layout/sheet_collection.xml` (lines 113-163):

- Dropped the `collectionTabs` LinearLayout wrapper.
- Added `androidx.constraintlayout.widget.Guideline` at 50% vertical.
- Made `tabList` and `tabBadges` direct children of the outer ConstraintLayout — each spans half the width via the guideline (`tabList` start→parent + end→guideline; `tabBadges` start→guideline + end→parent). Background `#16151B` moved from the wrapper onto each tab. Used `android:foreground="?attr/selectableItemBackground"` so the click ripple sits over the dark background instead of replacing it.
- `tabIndicator` now constrains to `@id/tabList` start/end — a real sibling — so both XML lint and the runtime `showTab()` code resolve correctly. Tab clicks should actually slide the underline on device for the first time since S275 once the next install lands.
- `collectionEmpty` / `collectionList` / `collectionGrid` top constraints already referenced `@id/tabIndicator` (a sibling) — no change.
