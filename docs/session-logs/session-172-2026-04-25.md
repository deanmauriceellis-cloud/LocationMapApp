### WickedAnimationOverlay built (S172 interim bridge)
- New file: `app-salem/src/main/java/com/example/wickedsalemwitchcitytour/wickedmap/WickedAnimationOverlay.kt`
- Subclass of `org.osmdroid.views.overlay.Overlay`. Each `draw()` call:
  - Builds a vanilla `CameraState` from `mapView.mapCenter` / `zoomLevelDouble` / width / height (osmdroid is Web-Mercator with TILE_SIZE 256, matches `MercatorMath`).
  - Forwards `tick(now)` + `draw(canvas, camera)` to each wrapped `MapOverlay`.
  - Calls `mapView.postInvalidateOnAnimation()` so the MapView keeps redrawing while overlays are alive.
- Rotation note: under heading-up rotation, particles are projected in the unrotated mercator frame and drawn on osmdroid's pre-rotated canvas. Particles rotate with the map (acceptable; geographic anchor preserved). Documented in the file header.

### Water animation iteration log (the long arc)
This is the load-bearing context for next session — water visual is still NOT what operator wants. Tuning continues.

1. **v1 — particle whitecaps** (initial S171 design). Curved arc strokes per particle, sin² intensity, ~6000 particles. Operator: "ocean doesn't look very animated."
2. **Density bump** (cap 500→1500). ANR (UI thread overload from particle iteration).
3. **v2 — wave bands** sweeping across each polygon. Operator: "I don't want the animation to be global, the white crests should be localized."
4. **v3 — localized windrow zones** (small patches per polygon). Operator: "very coarse, not very professional ... I need something simple and stupid."
5. **v4 — pulsing white dots** (FireflyOverlay clone, white). Operator: "the whitecaps look like fireflies, that is NOT what white crests are!"
6. **v5 — wave-crest curved strokes per anchor.** Each spot is a short `~` shape via `quadTo`. Density formula problem: per-polygon cap of 80 spread across Atlantic Ocean polygon (0.148 deg² bbox) = invisible at z18.
7. **v6 — screen-space pulsing-dot grid** clipped to water polygons. Density was right but operator: "moving on the map still shows them not being aligned to the map" (dots stayed at fixed screen positions while basemap panned underneath).
8. **v7 — world-anchored grid (lat/lon scan, filter inside-water at init)**. Initial 33m grid → 700k anchors → ANR during init. Coarsened to 110m grid → 14185 anchors built in 1.4s. Skips Atlantic Ocean polygon (bbox > 0.01 deg² excluded).
9. **Per-anchor duty cycle** (1.5s active / 14.5s idle, then 2.5s/8s) — was: continuous sin pulse made coverage feel "global"; bursting feels more natural.
10. **Position jitter** (random offset within each grid cell) — was: perfect grid pattern was visible at a glance.
11. **Final this session**: 14k jittered world-anchored anchors, 31% duty, wavy `~` strokes 13-19px, operator's verdict: **"not the best"**. To be continued S173.
