## Phase 9P.B Step 9P.7 — Admin route + AdminLayout shell — DONE

### Investigation
- Web app currently has no router and no `react-router-dom` dep. `main.tsx` renders `<App />` directly.
- Two routing options: (a) install `react-router-dom`, (b) path-based dispatch in `main.tsx`. Chose (b) — public web app has no other routing needs and the admin tool is a single screen, so an extra dep is unjustified weight. Vite's dev server falls back to `index.html` for unknown paths so `/admin` "just works" in dev/preview.
- Vite proxy: `/api` → `http://10.0.0.229:4300` rewriting `/api` prefix off. So `/api/admin/salem/pois` from the browser hits `/admin/salem/pois` on the cache-proxy, same-origin from the browser's POV — Basic Auth credentials are cached transparently.
- No existing `web/src/admin/` directory; created it.
- Tailwind is the styling system; `dark` class is the dark mode toggle on the public app. Admin tool stays light-mode for now (no spec for dark mode in 9P.7).

### Implementation
- **`web/src/admin/AdminLayout.tsx`** — NEW (~140 lines)
  - Header (slate-800, h-12): "LocationMapApp Admin / Salem POIs" title; **[Highlight Duplicates]** stub (slate, logs `console.log('… wiring lands in 9P.11')`); **[Publish]** stub (emerald, logs 9P.13); flex spacer; **Oracle: —** pill (slate-700 rounded-full, `title` attr documents 9P.10b wires it); **[Logout]** button.
  - Logout uses the **XMLHttpRequest 401 trick**: synchronous XHR to `/api/admin/salem/pois` with deliberately wrong `logout:wrongpassword` credentials in `xhr.open()`. The browser caches the new (wrong) credentials for the origin, replacing the good ones; the next admin API call therefore re-prompts. After the XHR, `window.location.reload()` forces a fresh render so the prompt appears immediately. Wrapped in try/catch because some browsers throw on deprecated sync XHR.
