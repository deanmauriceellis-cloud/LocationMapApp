## Admin-tool tooling shipped

1. **`historical_narration_needs_refinement` lint check.** Fires for
   self-ref / modern-leak / subject-absent / tag-echo / address-list
   patterns. 89 POIs flagged. Soft patterns (rambling/filler) deliberately
   not surfaced — those are draft-quality, not broken. Surfaces in the
   existing Lint tab with Open Editor / Suppress / Show on Map.
2. **Shared validator module** at `cache-proxy/lib/historical-narration-validator.js`
   used by the lint check, the scoring script, and (TS-mirrored) the
   variant picker.
3. **Historical Narration field always visible in PoiEditDialog.** Removed
   the `currentCategory === 'HISTORICAL_BUILDINGS'` gate so any POI can
   have one authored without a category change first.
4. **POI tree color indicators.** Each POI row gets a colored dot if it
   has a lint flag (red=error, amber=warn, sky=info). Categories /
   subcategories aggregate descendants — show highest severity dot, count
   column splits "flagged / total" when any descendants are flagged.
5. **Salem Oracle variant picker** at `web/src/admin/OracleVariantPicker.tsx`.
   Generates 5 short / 5 long / 5 historic variants in parallel via
   SI's `/api/intel/chat` (concurrency=3 to avoid GPU contention). Click
   a card to copy that text into the form field. Historic column hidden
   for POIs that aren't historical-eligible. Historic variants run
   through the validator — those that hit a critical pattern show
   disabled with reasons.
