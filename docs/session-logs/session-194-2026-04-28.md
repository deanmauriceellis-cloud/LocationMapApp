## Operator side-note: commemoratives ARE historical

Late in the session operator clarified that statues / memorials /
plaques / arches that pay tribute to PRE-1860 Salem history ARE
eligible for HISTORICAL_BUILDINGS classification AND historical_narration —
contradicting the S192 commemorative skip rule. The narration's subject
should be the figure / event commemorated, not the post-1860 object
(Hawthorne statue narrates Hawthorne's 1804-1864 Salem connection, not
the 1925 bronze). Counter-example: Bewitched / Samantha statue stays
non-historical because it commemorates 1960s TV.

Saved as `feedback_commemoratives_are_historical_tributes.md` and
indexed in MEMORY.md. Implementation deferred to next session — needs
a dedicated `buildCommemorativeTributePrompt()` in the generator with
a relaxed year guard, plus a "Commemorative tribute" historic variant
in the picker. A carry-forward comment was added in
`generate-historical-narrations.js` so next session picks it up at the
right spot.

I started implementing the generator path (replaced the skip with an
`isCommemorativeTribute` flag, exempted year-guard for tribute mode)
but reverted those edits when operator said "next session we can work
with that" — leaving the generator half-modified would have caused a
regression (commemoratives slipping through and getting the building
prompt). Net code change in the generator: just the carry-forward
comment.
