## Phase 3 — Auto-categorization via SalemIntelligence

Operator: "can we automate this? — we can use SalemIntelligence for hints."

**Coverage check.** 974/1188 (82%) of the missing-subcategory backlog already has `intel_entity_id` set — direct lookup against SI's `/api/intel/poi-export`. Remaining 214 fall to fuzzy match (200m proximity + name token overlap).

**Mapping engine** (`cache-proxy/lib/admin-suggest-categorization.js`):

- **SI primary → LMA (cat, sub) map** for the 20 distinct primary categories SI uses (restaurant, cafe, bar, fitness, hotel_lodging, spa_beauty, shop_retail, shop_occult, shop_bookstore, tour_operator, museum, service_professional, service_health, education, religious, gallery_art, performance_venue, etc).
- **Refiner functions** for ambiguous primaries — `shop_retail` keys off name+secondary+specialties (jewelry, florist, cannabis, tattoo, hardware, etc); `service_professional` routes by name (law, real estate, insurance, tax, otherwise companies); `service_health` (dentist, vet, pharmacy, hospital, nursing, clinic, otherwise doctors); `education` (library, childcare, kindergarten, university, otherwise schools); `tour_operator` (haunted, historical, night, otherwise walking).
- **Name-override layer** runs BEFORE SI mapping — catches definitive name keywords regardless of SI labeling. Auto/automotive/garage/tires/transmission/oil change → AUTO_SERVICES with sub. Self storage / U-Haul → SHOPPING__storage_rentals. Cycles/bicycle → SHOPPING__bicycle_shops. Catches SI-miscategorized rows like AutoZone (SI says service_professional) and Park-Tower-style holding companies.
- **Confidence tiers**: 1.00 direct + intel_id + same cat → auto-apply. 0.95 name override → auto-apply if same cat. 0.90 refined + intel_id + same cat → auto-apply. 0.70 cross-cat proposal → review only. 0.6 / 0.4 fuzzy → review only.
- **Auto-apply rule**: confidence ≥ 0.9 AND `proposed_category === current_category`. Never silently changes a category — only fills missing subcategory.

**Endpoints**:
- `POST /admin/salem/auto-categorize` — body `{ dry_run?: bool }`. Builds proposals, optionally applies eligible. Returns `{ total_proposals, auto_applied, auto_eligible_count, review_count, review[] }`.
- `POST /admin/salem/apply-categorization` — body `{ changes: [{ poi_id, proposed_category, proposed_subcategory }] }`. Operator-confirmed batch.
