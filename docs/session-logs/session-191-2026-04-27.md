## Investigation — is /move dropping is_tour_poi?

Verified the move path is clean:

- `cache-proxy/lib/admin-pois.js:455-465` — `/move` updates only
  `lat, lng, admin_dirty, admin_dirty_at, updated_at`. Never touches
  `is_tour_poi`.
- No triggers on `salem_pois` (`information_schema.triggers` empty).
- No back-end script flips `is_tour_poi` during move/edit/dedup.
- PoiEditDialog PUT sends only RHF `dirtyFields` — flag isn't touched
  unless the operator explicitly toggles the checkbox.

**Real client-side race found**: `PoiEditDialog.tsx:464-468` passes
`values: defaultValues` to `useForm`. When the operator drags the marker,
`AdminLayout.handlePoiMoved` rebuilds `selectedPoi.poi` (new ref → new
`defaultValues` ref → RHF resets the form). Any *unsaved* checkbox toggle
in the open dialog is wiped silently. Saved flags are unaffected.

Most likely cause for the 77 missing flags: S186's heuristic picked 216
POIs to bulk-flag and didn't blanket-cover the curated `HISTORICAL_BUILDINGS`
rows. They've been `false` since import.

## Hidden-POI narration bug — investigation (carry-forward to S192)

Operator: "check the android logs, see if our POI layers/choices match the
narrations, seems hidden POIs are still announcing"

Confirmed: **1,155 POIs are hidden on the map but flagged narrated**
(`default_visible=false` + `is_narrated=true`). They include 489 hist
buildings, 197 OFFICES, 147 HEALTHCARE, 133 CIVIC, 82 SHOPPING, 52
AUTO_SERVICES — modern businesses that have no business announcing during
a Salem tour.

**Root cause:** `app-salem/.../SalemPoiDao.kt:43` —
`findNarrated()` query is:

```kotlin
@Query("SELECT * FROM salem_pois WHERE is_narrated = 1 …")
```

It gates only on `is_narrated`, never on `default_visible`. So the
narration manager loads all 2,169 narrated POIs into its `points` list and
fires ENTRY events for any of them within geofence radius, regardless of
whether the user has them hidden via Layers checkboxes or
`default_visible=false`.

`NarrationGeofenceManager.isHistoricalQualified()` returns `true` for
everything when neither `tourMode` nor `historicalMode` is on — Layers
checkboxes only gate narration when tourMode is active (S186). In explore
mode, all loaded points narrate.

**Live evidence — last 10 narrations from device prefs**
(`/data/data/com.destructiveaigurus.katrinasmysticvisitorsguide/shared_prefs/narration_narrated_at.xml`):

| name | category | visible | tour |
|------|----------|---------|------|
| Jeff Goldman Immigration | OFFICES | **f** | f |
| JLP Construction Services Inc. | CIVIC | **f** | f |
| Muggeo & Associates | OFFICES | **f** | f |
| OctoCog Marketing & Design | OFFICES | **f** | f |
| Salem Farmer's Market | SHOPPING | t | f |
| Salem Firearm Safety | OFFICES | **f** | f |
| Salem Main Streets | CIVIC | **f** | f |
| Salem on Foot Tours | ENTERTAINMENT | t | f |
| Satanic Salem Walking Tours | ENTERTAINMENT | t | f |
| Yin Yu Tang | HISTORICAL_BUILDINGS | t | t |

6 of 10 are hidden-but-narrated. Salem Firearm Safety is also
`is_narrated=false` yet it ended up in the prefs file — that's likely
because the persistent mirror was hydrated from before the row was
flipped, or because it was triggered by reach-out logic. Either way the
visibility gate is the right place to enforce.

**Proposed S192 fix (not applied yet — operator session-ended before
green-lighting):**

```kotlin
@Query("SELECT * FROM salem_pois WHERE is_narrated = 1 AND default_visible = 1 …")
suspend fun findNarrated(): List<SalemPoi>
```

That makes `default_visible` the master kill switch for both map markers
and narration. Edge cases to consider:
- 55 POIs are `is_tour_poi=true` AND `default_visible=false` — those would
  go silent under this change unless the tour-mode override path is
  preserved. Probably want `is_tour_poi OR default_visible` rather than
  AND-ing default_visible alone.
- Layers checkboxes (POIs Hist Landmark / POIs Civic) currently only gate
  narration in tour mode. Whether explore mode should ALSO gate on Layers
  state is a separate decision the operator will need to make.
