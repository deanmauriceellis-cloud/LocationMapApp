### PoiTree refactor (`web/src/admin/PoiTree.tsx`)

Two new props, no behavior change for existing single-pane usage:
- `onDataLoaded?: (byKind) => void` — fires once after the initial fetch completes successfully so AdminLayout can hoist the dataset
- `externalByKind?: Record<PoiKind, PoiRow[]> | null` — when provided, the tree renders from this snapshot instead of its own internal copy. Lets AdminLayout patch a row's lat/lng (e.g. after a successful drag-to-move) and have the tree re-render off the new state without a re-fetch.

Internal `effectiveByKind = externalByKind ?? byKind` threads through `treeData` memo, `visibleCounts` memo, and the loading-state branch in the render. Internal fetch is preserved so the tree still works standalone. `useEffect` deps still `[]` (one-time mount load) — added an `eslint-disable-next-line` comment explaining `onDataLoaded` is intentionally excluded.

### `web/src/admin/AdminMap.tsx` — NEW (~370 lines)

Component layout (top to bottom of the file):

1. **Constants** — `SALEM_CENTER = [42.5197, -70.8967]`, `INITIAL_ZOOM = 14`, `KIND_COLORS` (tour=`#dc2626`, business=`#2563eb`, narration=`#059669`), `KIND_LABELS`, OSM tile URL.

2. **`poiIcon(kind, selected)`** — factory for `L.divIcon` with inline SVG circle. Cached by `${kind}|${sel|norm}` so we don't allocate fresh icons per marker. Selected variant: 18px (vs 12px), gold stroke (`#facc15`), 3px stroke width.

3. **`AdminMapProps` type** — `byKind | null` (full dataset from AdminLayout), `selectedPoi | null` (driven by both tree clicks and marker clicks), `onPoiSelect`, `onPoiMoved(kind, id, lat, lng)`.

4. **`PendingMove` type** — `{kind, poi, from, to, marker}`. Marker reference is kept so cancel can call `marker.setLatLng(from)` to revert.

5. **`<MarkerLayer>`** — child of `<MapContainer>`. Imperative cluster management:
   - Builds `L.markerClusterGroup({showCoverageOnHover: false, spiderfyOnMaxZoom: true, disableClusteringAtZoom: 18, maxClusterRadius: 50})` once per dataset change
   - Iterates `byKind` for tour/business/narration in order, skipping `deleted_at != null`, validating `lat`/`lng` are finite numbers
   - Each marker: `draggable: true, autoPan: true, title: "{Kind}: {name}"`, click handler emits selection, dragstart captures origin, dragend computes delta and emits PendingMove (no-op moves where lat/lng unchanged are ignored)
   - `markersByKeyRef` (`Map<"kind:id", {marker, kind}>`) lets us recolor a single marker on selection change without rebuilding the whole cluster
   - Cleanup on unmount/dataset change removes the layer from the map
   - **Separate effect** for selection change: `previousSelectedKeyRef` tracks the previously-highlighted key, the effect recolors the previous marker back to normal and the new one to selected. No cluster rebuild.

6. **`<FlyToSelected>`** — child of `<MapContainer>`. Watches `selectedPoi`, calls `map.flyTo([lat,lng], max(currentZoom,17), {duration: 0.6})` when the key changes. `lastIdRef` prevents flying twice on the same selection (e.g. when AdminLayout re-renders for unrelated state changes).

7. **`metersBetween(a, b)`** — Haversine for the confirm modal's distance display.

8. **`<MoveConfirm>`** — overlay div (z-index 500) over the map. Shows kind, name, id, from coords (6 dp), to coords (6 dp), distance in meters (1 dp). Cancel and Confirm buttons. Confirm shows "Saving…" + disables both buttons during the fetch. Errors render in a rose-50 box below the coords.

9. **`<Legend>`** — top-right corner overlay (z-index 400). Shows the three kind colors + counts.

10. **`<AdminMap>`** top-level — owns `pending`, `moveBusy`, `moveError` state. `selectedKey` derived from `selectedPoi`. `counts` memoized from `byKind` (visible/non-deleted). `handleDragEnd` → `setPending(move)`. `handleCancel` → marker.setLatLng(origin) + clear pending. `handleConfirm` → `fetch('/api/admin/salem/pois/{kind}/{id}/move', {method: POST, body: JSON.stringify({lat, lng})})`, on success calls `onPoiMoved`, on failure reverts the marker and shows the error in the modal. Loading state: shows "Loading POI map…" until `byKind` is non-null.
