### Post-commit: GitHub push rejected — switched to gitignore strategy

First push attempt failed: `remote: error: File app-salem-tiles-pack/src/main/assets/salem_tiles.sqlite is 261.61 MB; this exceeds GitHub's file size limit of 100.00 MB`. The blob had been pushed historically at the old path, but GitHub's pre-receive hook treats a new path as a new file. Operator chose option A via AskUserQuestion: **gitignore the file + document the regen path** (cleaner long-term than LFS — 262 MB generated artifact doesn't belong in git regardless).

Executed: `git reset --soft HEAD~1` to uncommit but keep changes staged, `git rm --cached app-salem-tiles-pack/src/main/assets/salem_tiles.sqlite` to untrack but leave on disk, updated `.gitignore` to exclude both the asset-pack path and the existing `tools/tile-bake/dist/salem_tiles.sqlite` staging path, documented the regeneration procedure in ASSETS-MANIFEST.md (run `tools/tile-bake/merge-into-bundle.js` then copy `dist/salem_tiles.sqlite` into `app-salem-tiles-pack/src/main/assets/`). Re-committed.
