## Item 2 explained: 307/364 lockout

The runtime gate `findNarrated()` in `SalemPoiDao.kt` requires
`is_narrated=1 AND (default_visible=1 OR is_tour_poi=1)`. The MHC-imported
hb_* POIs that S193 backfilled with `historical_narration` came in
hidden by design (`default_visible=0`, `is_tour_poi=0`), so the
geofence engine never queries them.

## Layers menu re-scoped (operator direction)

Operator clarified the Layers menu (POIs Hist. Landmark / POIs Civic)
should behave as **global overrides** — not tour-mode-only widenings as
S186 implemented. New semantics:

- **POIs Hist. Landmark ON** → POIs flagged `is_historical_property`
  speak regardless of tour mode. Prefer `historical_narration`; fall
  back to `short_narration` when missing. Long_narration stays excluded
  (matches `feedback_narration_fields_purpose_separated` HIST_BLDG
  runtime exception).
- **POIs Civic ON** → POIs flagged `is_civic_poi` are eligible for
  historical narration when `year_established <= 1860` OR they're a
  commemorative tribute (S194 rule).

`is_historical_property` already exists on `salem_pois` (NOT NULL bool,
default false), is in admin POI_UPDATABLE, has a checkbox in
PoiEditDialog ("Historical property — Listed or significant historic
site"), but is **populated on 0 rows**. That's the lever.
