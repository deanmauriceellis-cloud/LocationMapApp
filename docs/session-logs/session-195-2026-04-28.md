## Quality flags inline in PoiEditDialog (operator request)

When a POI is opened in the admin editor, the operator wanted to see
the lint warnings firing on it AND a way to suppress (override) them
without leaving the editor.

Implementation — frontend only, no backend changes:
- New "Quality flags" panel between the dialog header and the tabs.
  Renders one row per active lint check, with severity badge, label,
  category, message, fix hint, and a per-row Suppress button.
- Empty state: "No active quality flags on this POI." (emerald banner)
- Suppress posts to existing `POST /admin/salem/lint/poi/:id/suppress`
  with `{ check_id }`, then removes the row from local state.
- After suppress, dispatches a `salem-lint-changed` window event;
  PoiTree listens and re-fetches the per-POI flag map so the colored
  dot disappears immediately.

Net coupling: zero new props on PoiEditDialog (it talks straight to
the lint endpoint), one event listener added in PoiTree. TS clean
(`tsc --noEmit -p .` exit 0).
