## Pipeline finding

Initial investigation surfaced that **the existing import pipeline didn't match the operator's bio cohort or content shape:**

- `import-witch-trials-npc-bios.js` had been reading `~/Development/Salem/data/json/npcs/` and concatenating 7 fictional game-narrative sections (`narrative.life_before_1692`, `emotional_arc`, `how_they_see_world`, etc.) into the `bio` column. That's where the 2486-word avg came from.
- A separate, parallel directory `~/Development/Salem/data/json/biographies/` (output of `cmd/genbiographies` Go tool) already held 241 newspaper-style retrospective biographies with a tight `tts_full_text` field at avg 278 words. Tier-1+2 figures matched the operator's 49-cohort by id.
- Newspaper + tile imports went through SalemIntelligence at `:8089` rather than reading the Salem Oracle JSON files directly. That intermediary didn't auto-pick up regenerated content.

Decision: regenerate content with new word caps via `salem-village:latest` (gemma3:27b, ~21GB on the RTX 3090), then bypass SalemIntelligence — read JSON files directly into PG.

## Generator prompt edits

## Civic-flag drift fix (operator-discovered, end-session)

Operator field test: M&M Contractors was narrating during a tour walk despite being recategorized from CIVIC to SERVICES. Diagnosis: `is_civic_poi=true` flag (the canonical narration gate per S186) hadn't been cleared when the category was changed.

Audit query: **179 POIs** had the same drift — flag still set, category recategorized away from CIVIC. Breakdown: PARKS_REC 67, SERVICES 50, EDUCATION 36, WORSHIP 19, FOOD_DRINK / SHOPPING / WITCH_SHOP / HEALTHCARE / ENTERTAINMENT 7. Same lint area as S196 carry-forward `civic_flag_mismatch`.

One-shot SQL: `UPDATE salem_pois SET is_civic_poi = false WHERE category != 'CIVIC' AND is_civic_poi = true AND deleted_at IS NULL` — UPDATE 179. Verify: 0 drift, 85 CIVIC-consistent.

Drift prevention wired in two places:

`cache-proxy/lib/admin-pois.js` `buildUpdateClause()` — when the request body changes `category` and does NOT explicitly set `is_civic_poi`, auto-append `is_civic_poi = (cat === 'CIVIC')` to the SET clause. Defense-in-depth: catches any API caller. Explicit `is_civic_poi` in the body short-circuits the auto-sync (operator override path preserved).

`web/src/admin/PoiEditDialog.tsx` — added a `useEffect` watching `currentCategory`. Whenever the category dropdown changes, `setValue('is_civic_poi', cat === 'CIVIC', { shouldDirty: true })`. Visually consistent for the operator as they edit.

Both directions: CIVIC → flag turns ON, anything else → flag turns OFF. Operator-confirmed expected behavior.
