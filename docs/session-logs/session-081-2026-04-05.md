## Satellite Tile Upgrade: USGS → Esri World Imagery

**Problem:** USGS National Map Imagery maxed out at zoom 16 — too distant for a walking tour app.

**Solution:** Switched to Esri World Imagery (`server.arcgisonline.com`):
- Max zoom 19 (vs 16) — street-level detail with individual buildings/cars visible
- High resolution over Salem, MA (urban area with excellent Esri coverage)
- No API key required
- Attribution: "Sources: Esri, Maxar, Earthstar Geographics, and the GIS User Community"

**Files modified:**
- `TileSourceManager.kt` — Replaced USGS URL with Esri World Imagery, updated USGS_MAX_ZOOM to 19
- `SalemMainActivity.kt` — Updated cinematic zoom cap from 16 to 19

**Test result:** Zoom 19 in downtown Salem shows individual rooftops, parked cars, street markings. Massive improvement.

## Tour JSON Files Created

Created self-contained tour definition files in `app-salem/src/main/assets/tours/`:

| File | Tour | Stops | Size |
|------|------|-------|------|
| `tour_essentials.json` | Salem Essentials | 14 | 22KB |
| `tour_explorer.json` | Salem Explorer | 20 | 30KB |
| `tour_grand.json` | Grand Salem Tour | 26 | 38KB |

Each file contains:
- Tour metadata (name, theme, description, distance, time, difficulty)
- Complete stop array with: order, POI coordinates, geofence radius, walking time/distance from previous stop, transition narration (TTS walking directions), short narration (TTS approach), long narration (TTS at-location), description, historical period, admission, hours, phone, website

These files are portable — any module can load a tour from a single JSON file without needing database access.
