## Phase 1 — Gut Socket.IO from `ChatRepository`

Pattern parallels S209's `DebugHttpServer` gut: keep the public surface, delete the network internals, log the rationale at the source.

**Removed:**
- `import io.socket.client.IO`
- `import io.socket.client.Socket`
- `import kotlinx.coroutines.suspendCancellableCoroutine`
- `import kotlin.coroutines.resume`
- `import okhttp3.Request` (only the realtime path named it; REST path uses inferred type)
- `private var socket: Socket?` field + `onMessageCallback`/`onTypingCallback` fields.
- All Socket.IO option building, event listeners, `IO.socket(...)`, `socket.connect/disconnect/emit/on/off`.

**Replaced with no-op stubs (signatures preserved so callers compile unchanged):**

```kotlin
suspend fun connect() { /* no-op: V1 ships zero-network */ }
fun disconnect() { /* no-op */ }
fun isConnected(): Boolean = false
fun setOnMessageListener(callback: (ChatMessage) -> Unit) { /* no-op */ }
fun setOnTypingListener(callback: (String) -> Unit) { /* no-op */ }
fun joinRoom(roomId: String) { /* no-op */ }
fun leaveRoom(roomId: String) { /* no-op */ }
fun sendMessage(roomId: String, content: String, replyToId: Long? = null) { /* no-op */ }
fun sendTyping(roomId: String) { /* no-op */ }
```

REST methods (`fetchRooms`, `createRoom`, `fetchMessages`) **left in place** for now — the operator's chosen scope was Socket.IO drop, not full ChatRepository nuking. They're gated upstream by the same `showChatDialog` early-return so they never fire on V1, and dropping the full REST surface (plus the hardcoded `http://10.0.0.229:4300` and the OkHttp-based `client` field) is a cleaner stand-alone session. **Carry-forward to S212+** as "ChatRepository REST drop."
