### Diagnosis: why Bit Bar doesn't render on the map

Operator stated Bit Bar wasn't visible on the device even with the Show All POIs FAB on. Pulled comparison data from PG:

| | bit_bar_1 (not rendering) | sweet_boba (rendering) |
|---|---|---|
| category | FOOD_DRINK | FOOD_DRINK |
| merchant_tier | 0 | 0 |
| default_visible | true | true |
| **is_narrated** | **false** | **true** |
| is_tour_poi / is_civic / hist_property | all false | all false |
| description / short_narration text | empty | empty |

**Root cause:** `PoiCache.findNarrated()` at `PoiCache.kt:95-99` returns `(isNarrated OR haunt) AND (defaultVisible OR overrides)`. The map's only marker pipeline (`SalemMainActivity.loadNarrationPointMarkers`) consumes this exact pool. Sweet Boba clears `is_narrated=true`; Bit Bar fails the pool gate entirely, never reaches the FAB widener.

Initial mis-advice to operator: "set Merchant tier=1, write narration text, flip Narrated." Operator correctly pushed back — Sweet Boba is also Tier 0 with empty narration text yet renders fine. Corrected single-step answer: **only `is_narrated=true` matters**; runtime template synthesis handles empty-text Tier 0 POIs in the geofence.

Operator's architectural follow-up: "I would think the flag 'Visible by Default' would be the controller, not narration." Validated as a real architectural smell — `default_visible` and `is_narrated` conflate two orthogonal concerns (marker rendering vs audio gating).
