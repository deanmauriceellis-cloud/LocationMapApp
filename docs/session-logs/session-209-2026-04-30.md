## Phase 2 — PolygonLibrary.unload() + onTrimMemory hook (W-H2)

**`PolygonLibrary.kt`** — added:

```kotlin
fun unload() {
    if (!loaded && byKind.isEmpty()) return
    val freed = byKind.values.sumOf { it.size }
    byKind.clear()
    loaded = false
    Log.i(TAG, "unloaded — freed index of $freed polygon entries")
}
```

Idempotent. Frees the `byKind` map and its inner lists. Underlying `WickedPolygon` objects survive only if the three animation overlays still hold them (`AnimatedWaterOverlay`/`FireflyOverlay`/`RollingGrassOverlay` each take a `private val polygons: List<WickedPolygon>` in their constructor).

**`SalemMainActivity.kt`** — added `import android.content.ComponentCallbacks2` + new override placed right after `onDestroy()`:

```kotlin
@Suppress("DEPRECATION") // TRIM_MEMORY_* deprecated in API 35; still dispatched on target API 34.
override fun onTrimMemory(level: Int) {
    super.onTrimMemory(level)
    if (level >= ComponentCallbacks2.TRIM_MEMORY_RUNNING_LOW) {
        DebugLogger.i("SalemMainActivity", "onTrimMemory(level=$level) — dropping PolygonLibrary index")
        PolygonLibrary.unload()
    }
}
```

Threshold rationale: `TRIM_MEMORY_RUNNING_LOW` is the standard "system is starting to feel pressure" signal and is the lowest value above ordinary backgrounding (`TRIM_MEMORY_UI_HIDDEN`/`TRIM_MEMORY_BACKGROUND`/`TRIM_MEMORY_MODERATE` are bigger-numbered, so the `>=` check fires on all critical levels too).

**Limitation acknowledged in code comment:** the active overlays still hold their slice of the polygon list, so this only frees the duplicate index. A "drop overlay too on critical pressure" extension is deferred to V1.1+.
