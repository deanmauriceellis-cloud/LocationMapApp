Operator picked option B (load local Tiger MA data, keep everything offline).

Loader steps:
- Configured `tiger.loader_platform`/`loader_variables`: staging_fold=
  `/home/witchdoctor/tiger-staging`, peer-auth declare_sect (no PGUSER/PGPASSWORD).
- Generated state-25 loader via `loader_generate_script(ARRAY['MA'], 'sh')`.
- Initial run loaded ma_addr (576K), ma_featnames (703K), ma_place, ma_cousub,
  ma_tract, ma_tabblock20, ma_faces, ma_zip_*. **ma_edges failed** to import.
- Root cause: `tiger.edges` parent has 5 pgrouting columns (source, target,
  length_m, walk_cost, walkable) added by salem-router. Inherited into
  tiger_data.ma_edges; loader_load_staged_data's auto-INSERT included them
  but staging table from shp2pgsql lacked them → "INSERT has more target
  columns than expressions."
- Fix: custom `load-edges-only.sh` ALTERs `tiger_staging.ma_edges` to add
  the 5 columns as NULL placeholders after each shp2pgsql, before
  loader_load_staged_data runs. Re-running…

`salem.edges` (used by `tiger.route_walking()`) is independent of
`tiger_data.ma_edges`, so dropping/re-importing edges does NOT break Salem
routing.

After full load: 576K ma_addr, 703K ma_featnames, 761K ma_edges, full

### Lint suppression — false positives (operator: "things like the lighthouse")

Operator note: "There will be false events - we will need to flag it.
Things like the lighthouse, etc." Lighthouse-style POIs (no street address
by design) will always trip multiple checks; need a way to mark a POI as
known-OK for a specific check so the lint stops surfacing it.

- New table `salem_lint_suppressions(poi_id, check_id, reason,
  suppressed_at)` PK (poi_id, check_id), FK on poi_id with CASCADE.
- Helper `suppressionClause(checkId, params)` in admin-lint.js inlines a
  `AND id NOT IN (SELECT poi_id FROM salem_lint_suppressions WHERE
  check_id = $N)` filter on every POI-side check.
- New endpoints:
  - `POST /admin/salem/lint/poi/:id/suppress` body `{check_id, reason?}`
    (idempotent ON CONFLICT DO UPDATE)
  - `DELETE /admin/salem/lint/poi/:id/suppress/:check_id`
  - `GET /admin/salem/lint/suppressions[?check_id=]` (list)
- Frontend: per-item Suppress button (rose-pink, prompts for reason), plus
  toggle in the check header to show suppressed entries with Unsuppress
  button. End-to-end smoke-tested.
