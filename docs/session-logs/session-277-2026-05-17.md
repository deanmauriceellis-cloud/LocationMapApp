### Exploratory thread — ambient ant sprites (parked)

Operator: *"I want the map to be animated, can we make 1000 little people walking across the city if we zoom that close - just silly graphics … this is a casino of tourists - our map should be animated - this is a golden rule of applications with mapping."*

Iterated through three reframings:
1. **First framing:** ~1000 walking sprites at high zoom. Confirmed feasible at the engine level; flagged density tradeoff.
2. **Second framing:** 64×64 individual sprites with tour-cohort mode (20-40 sprites trailing baked tour polylines), day/night density, October bump. I flagged the foreground/POI-marker competition at that size and offered prototype scoping.
3. **Third framing (operator settled):** "ants" — 8-16px tiny CGI dots, purely background motion. Much lighter scope. No cohort/tour-trailing needed; random walks along sidewalks are enough.

**Engine audit (Explore subagent):** WickedMap engine is already production-wired with 4 live ambient animations at 30fps heartbeat:

| Overlay | Cadence | File |
|---|---|---|
| AnimatedWaterOverlay (whitecaps) | 30fps, 2.5s/8s duty | `wickedmap/AnimatedWaterOverlay.kt` |
| FireflyOverlay (cemetery orbs) | 30fps continuous pulse | `wickedmap/FireflyOverlay.kt` |
| RollingGrassOverlay (park grass) | 30fps, 1.5s/14s duty | `wickedmap/RollingGrassOverlay.kt` |
| SpriteOverlay (haunt sprites near POIs) | 30fps, geofence-triggered | `wickedmap/SpriteOverlay.kt` |

Heartbeat is `WickedAnimationOverlay.kt:55-108` self-invalidating via `postInvalidateDelayed(33ms)`. Orchestrator is wired into `SalemMainActivity.kt:1442-1477` at overlay position 0 (drawn under markers — satisfies `feedback_basemap_priority_over_animation.md` automatically). `MapOverlay` is a 2-method `tick + draw` interface; adding ants is a 5th sibling, not new infrastructure.
