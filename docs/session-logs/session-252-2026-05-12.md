**Built this session:**
- `tools/tigerbase/` — full pipeline scaffold (gitignored sources/out)
- `scripts/download-tiger.sh` — pulls TIGER/Line PRISECROADS + PLACE + COUNTY + STATE + COASTLINE + gazetteer (single-state or `--all` for CONUS, 49 states / ~1 GB total)
- `scripts/bake-tigerbase.py` — in-house renderer (no Mapnik). Pillow + Shapely. 5 layers (water, boundaries, roads, places, labels) rendered to separate transparent WebP pyramids so they composite cleanly at serve time. ProcessPool + Linux fork-COW so workers inherit the loaded shapefiles without re-loading. Per-tile label collision dedup. Per-layer `LAYER_ENCODE` (quality=70-75 method=6 WebP — lossless was empirically larger, reverted).
- `scripts/pack-pmtiles.py` — packs a loose tile tree into a single PMTiles archive (eliminates 4 KB block overhead at CONUS scale).
- `scripts/recompress-existing.py` — re-encodes existing tiles with current LAYER_ENCODE (defensive: only replaces if smaller).
- `cache-proxy/lib/tigerbase-tiles.js` — new `/tigerbase/manifest` + `/tigerbase/:layer/:z/:x/:y` routes; tries PMTiles archive first, falls back to loose tree (lets us serve in-flight bakes).
- `web/src/admin/TigerBaseTab.tsx` — new admin tab, sidebar entry 🌎 TigerBase under Tooling. Leaflet with 5 stacked tile layers + per-layer toggles + scope picker.
- `docs/plans/tigerbase-conus-bake.md` — scope + standing-rules-respected doc.
