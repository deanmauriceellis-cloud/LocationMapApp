- `<TileProviderPicker>` only renders when there are >1 providers, so the single-option dropdown disappears from the corner of the map.

**Backend** (`cache-proxy/lib/admin-tiles.js`):
- `ALLOWED_PROVIDERS` reduced to `Salem-Custom` only — server rejects Mapnik / Esri requests.
- Added native dep `sharp@0.34+` for crop/resize.
- New `lookupExactTile(provider, z, x, y)` helper.
- New `overzoomFromAncestor(provider, z, x, y)` — walks UP the pyramid up to 8 levels (1024× scale). For each ancestor zoom `z-dz`, computes the matching ancestor `(ax, ay) = (x>>dz, y>>dz)`, extracts the sub-region `(subX, subY, subSize) = ((x % 2^dz) * 256/2^dz, …, 256/2^dz)`, resizes to 256×256 (lanczos3 for dz≤2, nearest for chunkier upscales).
- New `overzoomFromChildren(provider, z, x, y)` — walks DOWN up to 2 levels. Probes corner child first; if present, stitches the 2^dz × 2^dz children into a single tile via sharp `composite`. Used when zoomed out past what's baked.
- Route handler now: 1) exact lookup → if hit return buf with `X-Tile-Source: exact`; 2) check 500-entry LRU cache; 3) walk up; 4) walk down. Each result tagged with `X-Tile-Source: overzoom-up-{dz}` or `overzoom-down-{dz}` so map-state debugging is visible in browser devtools.
- 500-entry LRU caches resized PNGs in process memory (`OVERZOOM_CACHE`). Tile archive itself stays read-only.
