### Root cause of "walk doesn't work + no TTS + tours broken" on Bluestacks

Single bug, three symptoms. Smoking gun in the stream:

```
[47] E/NarrationMgr: TTS init failed: status=-1
[450] NarrationMgr: Enqueued: LONG_NARRATION — NPS Visitor Center (queue: 1)
[451] NarrationMgr: TTS not ready yet — deferring poi_narration_… (queue: 1)
[562] WALK-SIM: DWELL (TTS active) at step 7 — narrating=NPS Visitor Center …
```

Bluestacks has no Google TTS engine. `NarrationMgr` correctly defers when TTS isn't ready. BUT walk-sim's DWELL condition checks `currentNarration != null` rather than "TTS actually speaking" — so the deferred narration sets `currentNarration`, walk-sim DWELLs, deferred narration never plays (no engine), DWELL never exits except via the 180s safety cap. Walk-sim appears hung; tours appear broken; no TTS because no engine.

Recommended user-side fix: install Google TTS in Bluestacks (operator did) + set as preferred engine + install English voice data (operator couldn't find Android Settings in Bluestacks's stripped UI). Code-side fix (defensive — handles any device with TTS init failure, not just Bluestacks): DWELL should be guarded by `NarrationMgr.ttsReady`. Code fix deferred — was not the operator's priority.
