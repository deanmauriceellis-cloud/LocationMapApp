### Session arc — Katrina's Collection Phase 2 + Phase 3 SHIPPED

**Operator framing on entry:** "Let's figure out a good way to do fun ghost portraits — paper napkin view of things." Pointed at `~/Documents/MiscAttributes.ods`, a 4-column attribute table (Professions / expressions / oddities / hair). Goal: mix-and-match attributes to give each ghost a unique personality that can seed image generation, knowing that **SalemIntelligence2 in ~1 week** will redo all of this with deterministic per-building seeding — so V1 just needs random-roll personas that look distinct.

**Paper-napkin → seed JSON → batch pipeline:**

1. **Attribute seed.** Wrote `cache-proxy/data/ghost-personas-seed.json` + mirrored `~/Documents/MiscAttributes.ods` with operator's 4 columns expanded toward Salem 1692 / Halloween archetypes (Puritan Goodwife, Accuser Girl, Jailer, Sexton, Naumkeag Native, etc.; punchy alt-expressions; etc.). Started at 36 professions × 28 expressions × 30 oddities × 22 hair = ~664k combos.
2. **Pair convention.** Locked **A/B paired portrait** spec: image A is the ghost in a normal expression; image B is the SAME ghost looking at the viewer with a fourth-wall-break smirk; 2% runtime swap per render to "mess with their heads." Saved as memory `project_katrinas_collection_ghost_asset_pairing.md`.
3. **Frame model — false start then simplification.** First plan: 12 heraldic shield SVGs × 8 metal stroke colors with per-frame aperture masks. Operator overrode after the napkin sketch: *"forget about the coat of arms view — just settle for a frame that covers the edges of the box and make a bunch of different overlays with maybe an emblem at the top center. Keep this simple."* Collapsed to N=8 self-contained rectangular overlay PNGs, transparent interior, baked-in top-center emblem, no runtime color compositing.
4. **Category scope stubbed.** V1 = HISTORICAL_BUILDINGS only (107 POIs). Seed JSON `_eligible_categories` block lists `planned_future: [RESTAURANTS, SHIPS_AND_SERVICES, RETAIL, LODGING, ENTERTAINMENT]` so commercial-tier merchants extend without refactor.

**SD batch generation (Pass 1 → Pass 3 evolution):**

- **Pass 1** (txt2img A + txt2img B with same seed, expression word swapped): identity drift on B (Ghost 4 lighthouse keeper came back as a totally different older man). Watercolor aesthetic landed but ghost-ness didn't (read as humans in costume).
- **Pass 2** (txt2img A + full img2img B at denoise=0.45): identity rock-solid via A→B img2img but **background drifted more than face** — operator caught this immediately: *"we are getting more background changes than the ghost itself."*
- **Pass 3** (txt2img A + **face-only inpaint B** with elliptical mask + denoise tuned to 0.55, plus full style spine rewrite: Salem 1692 witch trials + Halloween animated cartoon Tim Burton/Hotel Transylvania, dropped "watercolor"): background bit-identical between A and B via inpaint, Salem atmosphere landed hard (pumpkins, autumn leaves, candlelit dusk, raven), but **face changes still too subtle** — operator: *"give much more ability for the 2nd image to change from the first."*
- **Pass 4 tune** (face mask expanded to cover mouth+jaw, denoise→0.75, dedicated face-focused B prompt with `({expression}:1.5)` weighting, vocabulary punched up — dropped "faint", added "wide", "locked eye contact", "open-mouthed"): smirks finally land. Some style drift toward Pixar/Disney on B at 0.75 but identity preserved by surrounding pixels being bit-locked.

**Operator approved Pass 4 as the V1 first-pass quality.** Pipeline: `generate-ghost-portraits.py --from-pg` runs all 107 HIST_BLDG POIs from PG (poi_id hashed to int32 for the SD seed; SHA1 → 32-bit deterministic), generates 214 PNGs total in 669s (~11 min). Then `--regen-b --denoise 0.75` regenerated all 107 B images in 461s reusing existing A files.

**Asset pipeline downstream:**

- `pack-ghost-portraits.py` downscales source PNG (768×768, ~143 MB) → app webp (384×384 q85, ~4.2 MB) — 97% reduction.
- Frame overlays: `generate-frame-overlays.py` generates 8 frame PNGs at 768×768 on bright pure green background, chroma-keys to alpha, forces a centered alpha-rectangle cutout (so portrait shows through even when SD ignored "transparent interior"), generates 8 emblems separately (SD won't reliably place small named objects), composites each emblem at top-center via PIL. After tighter chroma-key threshold + one regen of `gothic_wood_skull` (broken) + one regen of `brass_rivets_compass` (frame-in-frame with green band), all 8 frames are usable. Source 5.4 MB, packed 336 KB.
- `build-ghost-gallery.py` — one-shot HTML inspector at `~/AI-Studio/ghost-batch-v1/inspector.html` — full grid of 107 ghosts with toolbar toggles (A↔B, color↔greyscale, frame on/off) + per-tile click flip. Used to QA the full batch before wiring to Android.
