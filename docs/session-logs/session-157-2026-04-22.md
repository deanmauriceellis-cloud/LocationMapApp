### Problem

Need an in-app point-to-point walking-directions router for Salem. Zero external dependencies at runtime (V1_OFFLINE_ONLY rule). Must snap arbitrary lat/lon → nearest intersection, run shortest path, return polyline for osmdroid overlay.

### Data that makes this easy

`tiger.edges` already has topology pre-built:
- `tnidf` + `tnidt` — from-node / to-node intersection IDs
- `geom` MultiLineString
- `mtfcc` feature-class code (for walkability filter)
- `roadflg` — road vs. non-road edges

`massgis.massdot_roads` gives walking-safety attributes spatial-joined onto TIGER:
- `lt_sidewlk` / `rt_sidewlk` — binary sidewalk presence per side
- `shldr_lt_w` / `shldr_rt_w` — shoulder width (walking-safety proxy if no sidewalk)
- `speed_lim`, `num_lanes` — for cost penalties

PostGIS 3.6.2 + **pgRouting already installed** in the `tiger` DB. No extension work needed.

### Why walking-only is a big simplification (S157 scope narrow)

- **No one-way streets matter** — pedestrians walk both directions on every street.
- **No `salem-oneway-overrides.csv` needed.**
- **No `reverse_cost` column** — one cost per edge, symmetric.
- **No turn restrictions.**
- The single biggest gotcha of a routing pipeline (one-way data gap in TIGER/MassGIS) is **eliminated**.

| H-codes / R-codes / P-codes | ❌ | Hydrography / rail / boundaries — excluded by `roadflg` already |

### Cost formula (walking)

```
cost_seconds = length_m / 1.4                          # 1.4 m/s baseline
penalty = (lt_sidewlk == 0 AND rt_sidewlk == 0
          AND speed_lim >= 35)  ?  3.0  :  1.0         # bare-shoulder penalty
cost_seconds *= penalty
```

Tunable. Initial pass uses the table; can refine later by real walk times along the Heritage Trail.

### Runtime performance (estimated for Lenovo TB305FU)

- Dijkstra over 2,200 edges: **< 10 ms** single-threaded Kotlin
- A* with haversine heuristic: **< 5 ms**
- k-d-tree nearest-node lookup: **< 1 ms**
- Polyline materialization from path: **< 2 ms**

Total user-perceived tap→polyline: under 20 ms. Feels instant.
