### Turn 2 — Fix attempt 1: synthetic Speaking→Idle pulse

Hypothesis: every silent-drop path in `NarrationManager.enqueue()` (master-mute, ringer silent, POI/Oracle group mute) needs to emit a "completion" event so observers see the segment as done.

Patched `NarrationManager.kt` — added `emitSyntheticComplete(segment)` helper, swapped every `return` in the gate paths for `emitSyntheticComplete(segment); return`. First implementation: `_state.value = Speaking(segment); _state.value = Idle` synchronously.

Built + bundletool installed on both devices (initial DELETE_FAILED on `adb uninstall` because both devices had the package from S272 install; recovered via `am force-stop` → `pm clear` → retry uninstall per S272 carry-forward sequence). Cleared logcat, restarted tail.

Root cause: `MutableStateFlow` conflates rapid changes and dedupes by `equals`. The synchronous `Speaking → Idle` round-trip collapsed to no observable change when the prior value was already Idle.
