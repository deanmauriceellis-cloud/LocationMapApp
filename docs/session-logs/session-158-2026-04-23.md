### Pipeline delivered (all under `tools/tile-bake/`)

1. **planetiler** (89 MB JAR, single-file download) → produced `salem-vector.mbtiles` (7.2 MB) from fresh Geofabrik MA extract (OSM data 2026-04-22), OpenMapTiles schema, z0-z16 vector, bounds -70.958/42.475/-70.835/42.545.
2. **style-salem.json** — custom MapLibre-GL style, "Witchy Parchment" palette. Key design: NO style rules referencing the `poi` layer → all commercial amenity labels disappear from the render. Roads styled as parchment-on-brown casing; neighborhood/district labels use **Metropolis Bold** uppercase for witchy-gothic feel; water labels italic uppercase (cartographic convention); street labels Noto Sans.
3. **fonts/** — OpenMapTiles Fonts v2.0 GitHub release (171 MB, 57 font faces including Noto Sans, Metropolis, Open Sans, Roboto, PT Sans). Build-time only; NOT shipped with app.
4. **render-tiles.js** — Node.js script using `@maplibre/maplibre-gl-native` headless + `sharp` + `better-sqlite3`. Renders z14-z18 full Salem + z19 downtown → 10,644 PNG tiles → `salem-custom.mbtiles` (82 MB). ~53 seconds on CPU (RTX 3090 untouched per GPU-caution rule).
5. **merge-into-bundle.js** — merges `salem-custom.mbtiles` into `app-salem/src/main/assets/salem_tiles.sqlite` under new provider name `Salem-Custom`. Uses osmdroid's MapTileIndex key encoding + TMS-to-XYZ y-flip at merge time. Preserves Esri-WorldImagery (4,534 tiles) and Mapnik (2,599 tiles) rows untouched. Idempotent (drops existing Salem-Custom rows before insert).

4. **"Why two Hawthorne Blvds?"** — divided-boulevard OSM data (two oneway ways per physical boulevard) getting labeled twice. Fix: `["!=", "oneway", 1]` filter on major-road labels + bump `symbol-spacing` to 400.
5. **"I don't see cemeteries"** — style only drew `landuse class=residential`; everything else in `landuse` was invisible. Fix: add cemetery (slate-green + dashed outline), hospital, school/college/university, industrial, playground/pitch; add landcover sand (beaches) and wetland. Salem Willows + Charter Street + Old Burying Point now visible.
6. **"What POI layer did we drop?"** — clarified OpenMapTiles `poi` vector layer (commercial amenity labels, dropped) vs app's own POI marker overlay (unaffected). Offered narrow carve-out options; operator picked option 3 (cemetery + monument + place_of_worship labels). Added three label rules — all witch-trials-relevant historic markers now labeled.
7. **"Color NHL/NRHP/LHD buildings in purple"** — MHC Inventory data (`massgis.mhc_inventory`) not in OSM. Initial approach: spatial-join MHC points to `massgis.structures` → 1,711 historic building footprints, exported as GeoJSON, added as second source with tiered purple fill (NHL deep `#4B1D7A` / NRIND `#7B4FA0` / NRDIS+LHD lavender `#B799C7`).
8. **"Why doesn't the purple match the building underneath?"** — OSM building layer and MassGIS structures layer are different datasets with different footprint granularity. MassGIS aggregates institutional complexes into single polygons; OSM has them split. Fix: unified everything — exported ALL 39,408 Salem MassGIS structures with tier attribute, replaced the OSM building layer entirely. Now every building polygon comes from MassGIS, `tier` property determines color. Perfect alignment; 84× more building coverage (471 OSM → 39,408 MassGIS).
9. **"Parcel property lines got cut out?"** — after the S156 QGIS screenshot the operator referenced, parcel boundaries were in MassGIS L3 but never made it into the raster pipeline. Fix: exported 32,727 Salem parcels from `massgis.l3_parcels_essex`, encoded via tippecanoe into `parcels.mbtiles`, added thin muted-pink line rule at z16+.
10. **"We are just generating tiles — why does it cost more size?"** — explained tile-count (4× more tiles due to z14-z19 vs Mapnik's z16-z18) × bytes-per-tile (+32% from richer content) = ~5.4× bundle cost vs plain Mapnik.
11. **"Can we convert to something smaller than PNG?"** — WebP lossless via sharp. Re-encoded all 10,644 Witchy tiles: bundle dropped 254.5 MB → 208.2 MB (−46 MB, −18%). Witchy tiles went 15.2 KB/tile → 10.8 KB/tile, now smaller than the satellite PNG tiles (12.4 KB).

### Pipeline architecture (final state)

Everything lives under `tools/tile-bake/`:

```
Geofabrik MA OSM ──[planetiler]──► salem-vector.mbtiles (OSM roads/water/parks, z0-z16)
                                           │
massgis.l3_parcels_essex ──[tippecanoe]──► parcels.mbtiles (parcel outlines)
                                           │
massgis.structures JOIN mhc_inventory ──[tippecanoe]──► buildings.mbtiles (w/ tier tag)
                                           │
style-salem.json ──[maplibre-gl-native headless]──► salem-custom.mbtiles (WebP lossless)
                                           │
                                   ──[merge-into-bundle.js]──► salem_tiles.sqlite (provider='Salem-Custom')
```
