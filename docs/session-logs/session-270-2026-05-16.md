### Build — backend extension

`cache-proxy/lib/admin-burst-photos.js`:

- Added per-session `.kept.json` persistence (`{ kept: ['<base64url-id>', ...] }`) with safe path-traversal helpers (`sessionDirSafe`, `readKeptSet`, `writeKeptSet`).
- New routes:
  - `GET  /admin/burst-photos/sessions/:session/kept` → `{ kept: [...] }`
  - `POST /admin/burst-photos/sessions/:session/kept` `{ id, kept: bool }` → toggle + persist
  - `POST /admin/burst-photos/sessions/:session/commit-cull` → atomic: load `.kept.json`, move every undeleted `.jpg` NOT in kept set to `<session>/.deleted/` (and thumbnail counterpart to `.deleted/.thumbs/`), reset `.kept.json` to clean slate. Returns `{ kept, culled, errors }`.
  - `GET  /admin/burst-photos/devices` → parsed `adb devices` output as `[{ serial, state }]`.
  - `POST /admin/burst-photos/pull` `{ deviceSerial }` → SSE stream of stdout/stderr from `adb -s <serial> pull /sdcard/Pictures/WickedSalemRecon/ /tmp/lmasalem-stage/` then `python3 tools/pull-and-organize-burst-photos.py`. Each event is `{ stage: pull|organize|done|error, line?, summary? }`. Invalidates the list cache on completion.
- Added `const { spawn } = require('child_process')`.
- `req.body` available because `express.json({ limit: '1mb' })` is mounted on app at `server.js:30`.

No new routes registration needed — `admin-burst-photos.js` is already mounted at `server.js:267`.

### Build — frontend new tab

`web/src/admin/ReconTriageTab.tsx` (new, ~580 lines, single file):

- Top toolbar: session dropdown (sorted, with photo count) + Refresh + "📲 Pull from device" button.
- Header strip: live `kept / to cull / total` counter + Clear keep marks + "🗑 Commit cull (N)" button (rose, disabled when no session/empty/committing).
- Main view: responsive Tailwind grid (3 cols → 10 cols at xl) of `ThumbTile` components. Each tile uses an `IntersectionObserver` with 400px rootMargin to lazy-load `/admin/burst-photos/:id/thumb` and stays loaded after first reveal. Click = toggle keep (optimistic UI + rollback on failure). Double-click = open lightbox. Hovering shows the timestamp.
- Lightbox: full-size image via `/admin/burst-photos/:id/image`, EXIF panel (GPS lat/lon, altitude, speed, heading, track, fix method, camera Make/Model), prev/next + photo counter, Keep toggle. Keyboard: `←/→` nav, `K`/Space toggle keep, `Esc` close.
- Pull modal: lists `adb devices`, defaults to first ready device, "Run pull + organize" button kicks off SSE stream into a black terminal-styled log pane (color-coded by stage: pull=slate, organize=sky, done=emerald, error=rose). Close button enabled after `done` event refetches the session list.
- Confirm modal for commit-cull explains the .deleted/ recoverability and the kept-list reset.

All modals use `createPortal(..., document.body)` with `z-[2000]`/`z-[2100]` to escape the parent stacking context (same pattern as `BurstPhotosOverlay`).
