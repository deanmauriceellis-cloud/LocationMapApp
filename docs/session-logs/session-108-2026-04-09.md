## Graphics overlap analysis (latest screenshot)

Read `~/Pictures/Screenshots/Screenshot from 2026-04-09 02-57-28.png` (1360×670, the latest of four screenshots from the session). Showed the POI edit dialog open on a "Custom House" tour POI.

**Findings (confirmed by cross-referencing source code):**

1. **Tab row clipping at narrow effective viewport.** The `TabList` at `PoiEditDialog.tsx:591` was `flex gap-1 overflow-x-auto` with 7 tabs (General · Location · Hours & Contact · Narration · Provenance · Linked Historical · Danger Zone) inside a `w-[800px] max-w-[95vw]` panel. At max browser zoom, the effective viewport shrinks dramatically, the panel hits its `max-w-[95vw]` ceiling at ~400-500px wide, and the right-side tabs scroll off the edge invisibly. UX hazard: operator can't tell at a glance that 5 more tabs exist behind a horizontal scroll.

2. **Legend bleeding through dialog backdrop.** The Legend in `AdminMap.tsx:352` was `absolute top-2 right-2 z-[400]`. The dialog backdrop in `PoiEditDialog.tsx:537` was `fixed inset-0 bg-black/40` — 40% opacity black means underlying content is 60% visible. The legend showed through the backdrop at ~60% strength, which read as overlap rather than the intended dim-not-blackout effect.

3. **Header right-side cluster crowding** (NOT FIXED — outside app scope). The Mullvad VPN browser-extension badge sits in the same vertical band as the admin tool's right-side header buttons (Highlight Duplicates / Publish / Logout / Oracle pill). This is browser chrome bumping into app content; not fixable in app code without adding artificial right-padding to the header.

## Polish edits applied (3 lines, 2 files)

Operator authorized "you can do these things" → made the minimal polish edits.

| File | Line | Old | New |
|---|---|---|---|
| `web/src/admin/PoiEditDialog.tsx` | 591 | `flex gap-1 overflow-x-auto` | `flex flex-wrap gap-1` |
| `web/src/admin/PoiEditDialog.tsx` | 537 | `bg-black/40` | `bg-black/65` |
| `web/src/admin/AdminMap.tsx` | 352 | `z-[400]` | `z-[100]` |

**Rationale:**
- Wrap is the cleanest fix for the tab clipping. Tabs flow to a second row when the panel is narrow; no horizontal scroll, all 7 always visible.
- Backdrop opacity bump from 40% → 65% suppresses the legend (and any other map content) behind the modal cleanly. Still translucent enough to show context, opaque enough that the operator's attention is anchored on the dialog.
- Legend `z-[400]` → `z-[100]` correctly places it ABOVE the Leaflet map content but BELOW any modal stacking context. Side benefit: the move-confirm modal at `AdminMap.tsx:294` (`z-[500]`) now correctly suppresses the legend too. Did NOT bump the move-confirm modal's `bg-black/30` backdrop — flagged for the operator as a "say the word" follow-up if they want it tightened.
