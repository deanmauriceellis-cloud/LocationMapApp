## Topic 1 — Web admin fuzzy search + map auto-filter

**Operator request:** "When I search, I need fuzzy like 'bed' to find bed and breakfasts and when I use search, the map should automatically only show what is found in the fuzzy search."

**What was there before:** `PoiTree.tsx` had a search box that did a single-field substring on POI name only (`node.data.label.toLowerCase().includes(t)`). Map didn't filter against search at all — only category (clicked tree node) and lint id sets.

**Implementation:**

- `web/src/admin/PoiTree.tsx`
  - Exported `tokenizeSearch(term)` (whitespace-split lowercase tokens) and `poiMatchesQuery(poi, tokens, categories, subcategories)` for re-use across tree + map.
  - Multi-field haystack: `name + id + address + description + short_narration + long_narration + historical_narration + categoryLabel + subcategoryLabel + slug-form of category/subcategory IDs`. Slug-form turns `LODGING__bed_and_breakfast` → `lodging bed and breakfast`, so "bed" hits the subcategory label even when no POI's name contains "bed".
  - Multi-token AND match (every token must appear in haystack).
  - Replaced single-field `poiSearchMatch` with `makePoiSearchMatch(categories, subcategories)` factory used by react-arborist.
  - New `onSearchChange?: (term: string) => void` prop fires on every keystroke so AdminLayout can mirror.

- `web/src/admin/AdminLayout.tsx`
  - New state `poiSearchTerm`. Mirrors PoiTree via `onSearchChange={setPoiSearchTerm}`.
  - New `useMemo` that tokenizes the term, runs `poiMatchesQuery` over the full POI list, and produces `Set<string>` of matching ids (or null when search is empty).
  - Passes `searchIdFilter` + `searchIdFilterLabel` to the POI-view AdminMap (Tours-view AdminMap untouched — no search box in that view).

- `web/src/admin/AdminMap.tsx`
  - New `searchIdFilter?: Set<string> | null` prop intersected into `filteredPois` after categoryFilter / lintIdFilter.
  - Floating sky-blue pill top-right of map: `Search: "<term>" (N)` with the count.

`npx tsc --noEmit` clean. Confirmed via DB spot-check: "bed" → matches `morning_glory_bed_and_breakfast` (name) AND any future POI in `LODGING__bed_and_breakfast` subcat (label "Bed & Breakfast").
