Read `docs/plans/poi-passport-replaces-tour-stops.md` via Explore agent. Entered plan mode. Walked operator through the 7 open questions via AskUserQuestion dialogs (per `feedback_dialog_questions.md`). Operator decisions:

1. **Sequencing:** Ship full Passport in V1 (3-5 sessions).
2. **Visit log architecture:** POI-keyed only, not passport-keyed. Single `passport_visit(poi_id, …)` in `user_data.db`. Per-tour passports are filtered POI-id lists that render against the one global visit table. Business POIs later plug into the same single log.
3. **Filter knobs:** category multi-select + `require_historical_narration` boolean + `min_geofence_radius_m` int + `year_established` min/max range.
4. **Tour completion dialog:** repurposed as Passport summary dialog with "Open Passport" CTA.
5. **Stamp art:** filled vs hollow witch hat (one SVG, two states).
6. **Reset:** Settings entry inside PassportSheet overflow menu, confirm dialog, truncates `passport_visit`.
7. **Completion trigger:** when `count(passport_visits for tour) == count(tour_passport.poi_ids)` — auto-fire on transition (one-shot per session). Manual "End tour" tap fires the same dialog with partial copy.
8. **Entry point:** new witch-hat icon in `toolbar_two_row.xml` (slot between About and Speaker).

Spawned Plan agent for code-validation. Surfaced three real issues the proposal missed:
- **Two Room version bumps**, not one (`SalemContentDatabase` v19→v20 *and* `UserDataDatabase` v2→v3). Latter requires a hand-written `MIGRATION_2_3` — strict no-destructive lockdown — or paid users lose `poi_encounters` history on update.
- **No existing Settings UI** to slot Reset into. Lives inside PassportSheet's overflow menu.
- **`TourState.Completed` will never fire** after stops are deleted (today it depends on `completedStops + skippedStops == totalStopCount`). Replacement: visit-count vs passport-count comparison (operator's design).
- **Detour anchor coupling** at `SalemMainActivityDetour.kt:180` consumes `currentStopIndex` — needs replacement helper (`currentLegIndexForDetourAnchor`) before deletion.
- **Toolbar overflow risk** on portrait Pixel 8 with a 9th icon.

Final plan written to `/home/witchdoctor/.claude/plans/abstract-doodling-nest.md`. 5-session slice (S1 backend / S2 Room schemas / S3 sheet + visit tracking / S4 deletion + completion trigger replacement / S5 field validation). Operator approved.
