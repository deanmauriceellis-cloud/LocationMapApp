### 2026-04-18 (post-midnight) — Operator bug report: POIs interrupt each other

Operator reported: POI narrations preempt each other instead of queuing.

**Root cause:** `SalemMainActivityNarration.kt` lines 876–907 — `enqueueNarration()` was calling `tourViewModel.cancelSegmentsWithTag("poi_narration")` on every new POI ENTRY once the prior POI's 10-second min-hold had elapsed. That kills the in-progress `NarrationManager` segments and lets the new POI's `speakTaggedNarration` play immediately. Exactly the "POIs interrupt each other" symptom.

**Fix (this session):** Deleted the `priorPoiStartedAgoMs`/`priorPoiMinHoldMs` branch and the auto-preempt `cancelSegmentsWithTag("poi_narration")` + `currentNarration = null`. Preservation rules:
- Newspaper narration (`newspaper_1692` tag) still canceled — newspapers always yield to POIs. Intent preserved from Phase 9R.0.
- Explicit user-tap (`jumpToFront=true`) still preempts — "I tapped this POI to play it now" is honored.
- Auto-ENTRY (`jumpToFront=false`) with a prior POI playing falls through to the existing `else { narrationQueue.addLast(point) }` branch at line ~962. `currentNarration` gets cleared by the `NarrationState.Idle` observer at line 331 when TTS finishes; `playNextNarration()` then picks the next candidate from `narrationQueue` via `pickNextFromQueue()` (tier + distance selector), so the queue drain path is unchanged.

The two-layer queue (activity-level `narrationQueue` + `NarrationManager` internal `ArrayDeque`) already had all the plumbing to queue-and-serve. The bug was strictly the overzealous cancel in the activity layer.

**Secondary consequence:** the S125 `lastPoiNarrationStartMs` min-hold-stamp writes at the two narration-start sites are still valid (nothing reads the value anymore for the auto-preempt case, but the stamp is harmless). Left unchanged.

### 2026-04-18 — Operator bug report: historical POIs not taking priority over Oracle newspaper facts

**Symptom:** operator on the Lenovo reported that while walking past a historical POI, the 1692 Oracle newspaper narration keeps playing — the POI's narration doesn't cut in.

**Root cause:** race condition between activity-level `enqueueNarration` and the `NarrationState.Idle` StateFlow observer at `SalemMainActivityNarration.kt:341-346`. Flow:

1. Oracle newspaper is speaking (state=Speaking).
2. POI ENTRY fires → `enqueueNarration(poi)` on main thread:
   - Calls `tourViewModel.cancelSegmentsWithTag("newspaper_1692")` which cancels the current segment via `tts.stop()`.
   - `tts.stop()` eventually triggers `onError` on the TTS background thread → `NarrationManager` sets `_state.value = NarrationState.Idle` briefly.
3. Meanwhile main thread continues: sets `currentNarration = poi`, calls `speakTaggedNarration("poi_narration", ...)` → segments enqueue in `NarrationManager`, which pops the first one → `_state.value = Speaking` again.
4. The transient `Idle` emission from step 2 still gets observed. Its handler (line 343) unconditionally does `currentNarration = null`, stomping the value the main thread just set.
5. With `currentNarration = null`, subsequent silence-fill paths assume "nothing is playing" and race another newspaper into the queue ahead of the POI's body segment.

**Fix (this session):**

- Added `internal var narrationCancelForPoiAtMs: Long = 0L` to `SalemMainActivity`.
- In `enqueueNarration` (before the `cancelSegmentsWithTag("newspaper_1692")` call): stamp `narrationCancelForPoiAtMs = System.currentTimeMillis()`.
- In the Idle observer, at the top of the "Idle && narrationAutoPlay" block: if `System.currentTimeMillis() - narrationCancelForPoiAtMs` is in `[0, 500]`, log `Idle suppressed (transient cancel-for-POI N ms ago)` and `return@collectLatest` before touching `currentNarration`. The Speaking emission from the POI's segment will arrive shortly and properly drive the UI; the next legitimate `Idle` (after the POI finishes) will run the full cleanup.

Files modified:
- `app-salem/src/main/java/com/example/wickedsalemwitchcitytour/ui/SalemMainActivity.kt` — new `narrationCancelForPoiAtMs` field.
- `app-salem/src/main/java/com/example/wickedsalemwitchcitytour/ui/SalemMainActivityNarration.kt` — stamp in `enqueueNarration`, check + bail-out in the Idle observer.
