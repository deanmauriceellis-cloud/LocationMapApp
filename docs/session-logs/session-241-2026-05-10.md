### Implementation

**Backend** — `cache-proxy/lib/admin-mass-edit.js` (new, ~480 LOC):
- `GET /admin/salem/pois/export-spreadsheet` — query all 2070 live POIs, build workbook via SheetJS, write autofilter + hidden column A (UUID) + Export Info sheet (timestamp + column manifest). SheetJS community writer doesn't emit frozen-pane XML, so post-write `patchFrozenPane()` decompresses the .xlsx zip via JSZip, injects `<sheetView><pane ySplit="1" topLeftCell="A2" activePane="bottomLeft" state="frozen"/></sheetView>`, recompresses. Result: 1.05 MB .xlsx.
- `POST /admin/salem/pois/import-spreadsheet` — multer multipart upload to `cache-proxy/tmp/mass-edits/`, parse via XLSX.readFile, diff every editable cell against live PG row with kind-specific normalization (text/numeric/boolean/date/jsonb). Returns JSON `{ changeset: [{ poi_id, poi_name, stale, changes: [{column, kind, old, new, error?, warning?}] }] }`. Stale detection compares `pgRow.updated_at` against `Export Info.exportedAt`. Lat/lng outside Salem bbox surfaces as `warning` (frontend leaves checkbox unchecked by default). Temp file deleted post-parse.
- `POST /admin/salem/pois/apply-mass-edit` — group approvals by poi_id, coerce cell values, apply via existing `buildUpdateClause()` from admin-pois.js (refactored to be exported). Single PG transaction, rollback on any error.
- Editable column set: `UPDATABLE_FIELDS` minus `EXCLUDE_FROM_SPREADSHEET` = 60 cols. Excludes JSONB-complex (narration_subtopics, action_buttons, hours, building_footprint_geojson), corridor_points, and rarely-touched provenance refs.
