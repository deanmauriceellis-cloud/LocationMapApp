## Phase 9P.A.3 ‚Äî Migrate tour_pois + salem_businesses to PG ‚Äî DONE

### Strategy
- Use `better-sqlite3` to read directly from `salem-content/salem_content.db` (cleaner than the SQL-parsing pattern S98 used for narration; both libraries are already cache-proxy deps)
- Single Node.js script that handles both tables
- Type conversions: JSON TEXT ‚Üí JSONB; INTEGER 0/1 ‚Üí BOOLEAN; INTEGER Unix-millis ‚Üí TIMESTAMPTZ (with `0` ‚Üí NULL)

### First run failed cleanly
First attempt used `TRUNCATE TABLE` then `INSERT`. Error:
> cannot truncate a table referenced in a foreign key constraint

Found via `pg_constraint` query: six tables reference `salem_tour_pois` via FK ‚Äî `salem_historical_figures.primary_poi_id`, `salem_historical_facts.poi_id`, `salem_timeline_events.poi_id`, `salem_primary_sources.poi_id`, `salem_tour_stops.poi_id`, `salem_events_calendar.venue_poi_id`. All currently NULL (Phase 9Q backfill is future work) but PG still refuses TRUNCATE without CASCADE.

`salem_businesses` has zero FK constraints referencing it but I switched both to UPSERT for consistency.

### Fix
Switched both migrations to `INSERT ... ON CONFLICT (id) DO UPDATE SET ...` UPSERT pattern. Avoids the FK issue entirely and is fully idempotent ‚Äî re-running just refreshes rows.
