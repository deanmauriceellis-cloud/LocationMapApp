Picks the closest match within radius via JS-side haversine. First version did per-row `ST_Transform(the_geom, 4326)::geography` inside `ST_DWithin` which defeated the index — query timed out at 5 s. Rewrite uses the spatial index correctly: 5 nearest by name pattern, post-filter in JS.
