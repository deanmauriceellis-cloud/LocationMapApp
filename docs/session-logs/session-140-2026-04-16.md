### 2026-04-17 ‚Äî Bundle-script fix + Room DB re-bake

**Latent bug found:** `cache-proxy/scripts/bundle-witch-trials-into-db.js` uses `INSERT OR REPLACE` keyed on the primary-key `id`. When ids change (as they did here), old rows aren't overwritten ‚Äî they coexist. Pre-fix, running the bundle would have produced **32 rows** in the bundled SQLite: 16 stale S128 + 16 new Oracle, both under the same `tile_order` 1..16. Android UI would show 32 tiles, every tile_order doubled with mismatched content.

**Fix:** added `DELETE FROM salem_witch_trials_articles` before the `INSERT OR REPLACE` transaction, plus a pre-count log line so future bakers can see what was cleared. Matches the pattern already used by `publish-witch-trials-to-sqlite.js`. Small, scoped fix; no refactor.
